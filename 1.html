Chapitre 01 25 min Débutant

🎨 Introduction à CSS

Comprendre ce qu'est CSS et comment l'utiliser

🔍 1.1 Qu'est-ce que CSS ?

CSS signifie Cascading Style Sheets (Feuilles de style en cascade). C'est un langage qui décrit la présentation des documents HTML.

📖 Explication approfondie :

HTML s'occupe de la structure et du contenu (ce qui est affiché), tandis que CSS s'occupe de l'apparence et de la mise en page (comment c'est affiché).

🏠
Analogie : HTML est comme les murs et les pièces d'une maison (structure), CSS est comme la peinture, la décoration et l'agencement des meubles (style). Sans CSS, le web serait comme un monde en noir et blanc sans mise en page.

📜 Un peu d'histoire :

  • 1996 : CSS1 est créé par le W3C (World Wide Web Consortium)
  • 1998 : CSS2 apporte le positionnement et les médias
  • 2011 : CSS3 introduit les modules (Flexbox, Grid, animations, etc.)
  • Aujourd'hui : CSS continue d'évoluer avec de nouvelles fonctionnalités

📝 1.2 La syntaxe CSS

Une règle CSS est composée de trois parties :

CSS Syntaxe de base 
sélecteur {
    propriété1: valeur1;
    propriété2: valeur2;
    propriété3: valeur3;
}

Exemple concret :

h1 {
    color: blue;
    font-size: 24px;
    text-align: center;
    margin-bottom: 20px;
}

Explication détaillée :

  • sélecteur : l'élément HTML à styliser (h1, p, .classe, #id)
  • propriété : ce qu'on veut changer (color, font-size, margin)
  • valeur : comment on veut le changer (blue, 24px, center)
  • ; : chaque déclaration se termine par un point-virgule
  • { } : les accolades encadrent toutes les déclarations
Astuce : L'indentation (espaces au début des lignes) n'est pas obligatoire mais rend le code beaucoup plus lisible. Prenez l'habitude d'indenter proprement !

📥 1.3 Les trois façons d'inclure CSS

1. CSS externe (recommandé)

Fichier .css séparé lié à la page HTML

<!-- Dans le <head> du HTML -->
<link rel="stylesheet" href="style.css">

✅ Réutilisable sur plusieurs pages

✅ Plus facile à maintenir

✅ Cache navigateur (chargement plus rapide)

✅ Séparation claire HTML/CSS

2. CSS interne

Dans la balise <style> du head

<style>
    h1 {
        color: blue;
        text-align: center;
    }
</style>

✅ Utile pour une seule page

✅ Pas de fichier supplémentaire

❌ Non réutilisable sur plusieurs pages

❌ Alourdit le HTML

3. CSS en ligne (inline)

Directement dans l'attribut style de la balise

<h1 style="color: blue; text-align: center;">
    Titre
</h1>

❌ Mélange HTML/CSS

❌ Difficile à maintenir

❌ Non réutilisable

❌ Spécificité trop élevée

💡
Recommandation : Toujours utiliser un fichier CSS externe pour une meilleure organisation, maintenabilité et performance. Les CSS internes peuvent être utiles pour des prototypes rapides, et les styles en ligne sont à éviter sauf cas très spécifiques (ex: emails HTML).

🌊 1.4 La cascade et l'héritage

Cascade : L'ordre des règles CSS a de l'importance. Quand plusieurs règles s'appliquent au même élément, la dernière règle (ou la plus spécifique) l'emporte.

Héritage : Certaines propriétés sont automatiquement transmises des parents aux enfants (ex: color, font-family, line-height).

🔹 L'héritage en pratique :

/* CSS */
body {
    color: #1F2D3D;
    font-family: Arial, sans-serif;
}

/* Tous les éléments dans body héritent de ces propriétés,
   sauf si on les redéfinit spécifiquement */

p {
    color: #FF7A45;  /* Remplace l'héritage pour les paragraphes */
}

🔹 La cascade et la spécificité :

Quand plusieurs règles s'appliquent, le navigateur calcule un score de spécificité :

Sélecteur Spécificité Exemple
Styles en ligne1000style="color: red;"
ID100#monId
Classe, attribut, pseudo-classe10.maclasse, [type="text"], :hover
Élément, pseudo-élément1h1, p, ::before
 
/* Exemple de calcul de spécificité */
p { color: blue; }                /* Spécificité: 1 */
.texte { color: red; }              /* Spécificité: 10 (l'emporte sur 1) */
p.texte { color: green; }           /* Spécificité: 1 + 10 = 11 (l'emporte) */
#paragraphe { color: orange; }        /* Spécificité: 100 (l'emporte sur tout) */
Attention : Évitez d'utiliser !important qui force une règle et casse la cascade naturelle. C'est considéré comme une mauvaise pratique sauf cas exceptionnels.

📝 1.5 Les commentaires en CSS

Les commentaires sont ignorés par le navigateur et servent à documenter le code.

/* Ceci est un commentaire sur une seule ligne */

/*
    Ceci est un commentaire
    sur plusieurs lignes
    très utile pour expliquer
    des sections complexes
*/

.classe {
    color: blue; /* Commentaire en fin de ligne */
    margin: 10px; /* Explique pourquoi cette marge */
}

🎯 Bonnes pratiques :

  • Commentez les sections importantes de votre code
  • Expliquez les "pourquoi" pas les "comment" (le code parle déjà de lui-même)
  • Utilisez des commentaires pour organiser votre fichier :
/* ===== RESET ===== */
...

/* ===== HEADER ===== */
...

/* ===== BOUTONS ===== */
...

✍️ 1.6 Exercices pratiques

Exercice 1 : Premier style

Objectif : Créer une page HTML simple et lui appliquer du CSS externe.

Instructions :

  1. Créez un fichier index.html avec un titre h1, un sous-titre h2 et deux paragraphes
  2. Créez un fichier style.css et liez-le au HTML
  3. Changez la couleur du h1 en orange (#FF7A45)
  4. Changez la couleur du h2 en bleu foncé (#1F2D3D)
  5. Mettez les paragraphes en gris (#6B7280)
  6. Centre le h1
Voir la solution 
<!-- index.html -->
<!DOCTYPE html>
<html>
<head>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Titre principal</h1>
    <h2>Sous-titre</h2>
    <p>Paragraphe 1</p>
    <p>Paragraphe 2</p>
</body>
</html>

/* style.css */
h1 {
    color: #FF7A45;
    text-align: center;
}

h2 {
    color: #1F2D3D;
}

p {
    color: #6B7280;
}

Exercice 2 : Comprendre la cascade

Objectif : Expérimenter avec la cascade et la spécificité.

Instructions :

  1. Créez un paragraphe avec une classe "important" et un id "special"
  2. Définissez trois règles :
    • p : couleur bleue
    • .important : couleur orange
    • #special : couleur verte
  3. Observez quelle couleur l'emporte
  4. Ajoutez un deuxième paragraphe sans classe ni id

🎯 Les sélecteurs de base

Cibler les éléments HTML avec précision

🔤 2.1 Sélecteur d'élément (type selector)

Le sélecteur d'élément cible tous les éléments HTML d'un type donné.

h1 { color: #FF7A45; }     /* Tous les titres h1 */
p { font-size: 16px; }      /* Tous les paragraphes */
div { margin: 10px; }        /* Toutes les divisions */
a { text-decoration: none; } /* Tous les liens */

🎯 Utilisation typique :

  • Styles de base pour tous les éléments d'un type
  • Reset ou normalisation
  • Styles globaux

🔹 2.2 Sélecteur de classe (.)

Le sélecteur de classe cible tous les éléments ayant un attribut class spécifique. Une classe peut être réutilisée sur plusieurs éléments.

/* CSS */
.important {
    font-weight: bold;
    color: #FF7A45;
    font-size: 1.2em;
}

.carte {
    border: 1px solid #6B7280;
    padding: 20px;
    border-radius: 8px;
    background: white;
}

.texte-gris {
    color: #6B7280;
    line-height: 1.6;
}

<!-- HTML --> 
<p class="important">Ce texte est important et en orange</p>
<div class="carte">
    <h3>Titre de la carte</h3>
    <p class="texte-gris">Contenu de la carte en gris</p>
</div>
<p class="important texte-gris">Texte avec deux classes</p>

📖 Classes multiples :

Un élément peut avoir plusieurs classes séparées par des espaces. Cela permet de combiner des styles :

<div class="boite rouge gras bordure">
    Cette boite combine 3 classes
</div>
Bonnes pratiques :
  • Utilisez des noms de classes explicites (.menu-principal, .bouton-achat)
  • Évitez les noms basés sur l'apparence (.rouge, .gras) car le style peut changer
  • Préférez les noms sémantiques (.erreur, .alerte, .produit)

🔸 2.3 Sélecteur d'ID (#)

Le sélecteur d'ID cible un élément unique avec un attribut id spécifique. Un ID doit être unique dans toute la page.

/* CSS */
#en-tete {
    background-color: #1F2D3D;
    color: white;
    padding: 20px;
    position: sticky;
    top: 0;
}

#menu-principal {
    display: flex;
    justify-content: space-between;
    list-style: none;
}

#pied-page {
    background: #1F2D3D;
    color: #F6EDE7;
    text-align: center;
    padding: 15px;
}

<!-- HTML --> 
<header id="en-tete">
    <nav id="menu-principal">...</nav>
</header>
<footer id="pied-page">...</footer>
⚠️
Règle importante : Un ID ne doit être utilisé qu'une seule fois par page. Pour des styles réutilisables, privilégiez les classes. Les IDs sont plus adaptés pour :
  • Les ancres de liens (href="#section")
  • Le ciblage JavaScript (document.getElementById())
  • Les éléments uniques (header, footer, menu principal)

🌟 2.4 Sélecteur universel (*)

Le sélecteur universel cible tous les éléments de la page.

/* Réinitialisation des marges/paddings pour tous les éléments */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

/* Appliquer une police à toute la page */
* {
    font-family: 'Segoe UI', sans-serif;
}

/* Sélecteur universel dans un contexte spécifique */
.carte * {
    margin-bottom: 10px;  /* Tous les éléments dans .carte */
}

🎯 Utilisations courantes :

  • Reset CSS (réinitialiser les styles par défaut)
  • Appliquer des styles à tous les éléments
  • Sélectionner tous les enfants d'un conteneur
Attention : Le sélecteur universel peut être lent si utilisé abusivement. Évitez de l'utiliser dans des sélecteurs complexes comme .menu * li.

👥 2.5 Sélecteurs groupés

Les sélecteurs groupés permettent d'appliquer les mêmes styles à plusieurs sélecteurs en les séparant par des virgules.

/* Même style pour h1, h2, h3 */
h1, h2, h3 {
    font-family: 'Arial', sans-serif;
    color: #1F2D3D;
    margin-bottom: 10px;
}

/* Plusieurs classes */
.important, .urgent, .alerte {
    font-weight: bold;
    color: #FF7A45;
}

/* Mélange de sélecteurs */
h1, .titre-principal, #logo {
    text-transform: uppercase;
}

🎯 Avantages :

  • Code plus concis et plus facile à maintenir
  • Évite la duplication de code
  • Plus facile à modifier (un seul endroit à changer)

🌳 2.6 Sélecteurs descendants (espace)

Le sélecteur descendant (un espace) sélectionne les éléments qui sont à l'intérieur d'autres éléments, à n'importe quel niveau.

/* Tous les paragraphes dans un article */
article p {
    line-height: 1.6;
    margin-bottom: 15px;
}

/* Tous les liens dans le menu */
.menu a {
    text-decoration: none;
    color: #1F2D3D;
    padding: 10px;
}

/* Tous les éléments de liste dans une navigation */
nav ul li {
    display: inline-block;
    margin-right: 15px;
}

/* Plusieurs niveaux */
.sidebar .widget p a {
    color: #FF7A45;
}

🎯 Exemple HTML :

<nav class="menu">
    <ul>
        <li><a href="#">Accueil</a></li>  <!-- .menu a s'applique -->
        <li><a href="#">Produits</a></li> <!-- .menu a s'applique -->
    </ul>
</nav>
Astuce : Les sélecteurs descendants sont très utiles mais attention à ne pas les rendre trop spécifiques (ex: div.container .content article p span est difficile à maintenir).

👪 2.7 Sélecteurs enfants (>)

Le sélecteur enfant (>) sélectionne uniquement les éléments qui sont des enfants directs d'un parent.

Avec > (enfant direct)

.menu > li {
    border-bottom: 1px solid #6B7280;
}

S'applique uniquement aux <li> directement dans .menu

Sans > (descendant)

.menu li {
    border-bottom: 1px solid #6B7280;
}

S'applique à tous les <li> dans .menu, même dans les sous-listes

/* Exemple concret */
.menu > li {
    display: inline-block;      /* Uniquement les li directs */
    position: relative;
}

.menu > li > ul {
    display: none;               /* Le ul direct sous li */
    position: absolute;
}

.menu > li:hover > ul {
    display: block;              /* Affiche le sous-menu au survol */
}

🎯 HTML correspondant :

<ul class="menu">
    <li>Accueil</li>                                 <!-- .menu > li OK -->
    <li>Produits
        <ul>
            <li>Électronique</li>                    <!-- PAS .menu > li -->
            <li>Vêtements</li>                       <!-- PAS .menu > li -->
        </ul>
    </li>
</ul>

➕ 2.8 Sélecteurs adjacents (+ et ~)

Sélecteur adjacent direct (+)

Sélectionne l'élément qui est directement après un autre élément.

h2 + p {
    font-weight: bold;
    margin-top: 0;
}

Signification : Le paragraphe qui vient juste après un h2

Sélecteur général adjacent (~)

Sélectionne tous les éléments après un autre élément.

h2 ~ p {
    color: #6B7280;
    margin-left: 20px;
}

Signification : Tous les paragraphes qui viennent après un h2

/* Exemples concrets */

/* Style pour le premier paragraphe après un titre */
h1 + p {
    font-size: 1.2em;
    font-weight: 300;
    color: #FF7A45;
}

/* Tous les éléments après une erreur */
.erreur ~ * {
    opacity: 0.5;
}

/* Style pour les éléments alternés */
li + li {
    border-top: 1px solid #6B7280;
}

🎭 2.9 Pseudo-classes de base

Les pseudo-classes définissent l'état spécial d'un élément.

Pseudo-classe Description Exemple
:hover Quand la souris survole l'élément a:hover { color: #FF7A45; }
:active Pendant le clic (moment où on appuie) button:active { transform: scale(0.95); }
:focus Quand l'élément a le focus (input, lien) input:focus { border-color: #FF7A45; }
:visited Lien déjà visité a:visited { color: purple; }
:link Lien non visité a:link { color: blue; }
:first-child Premier enfant de son parent li:first-child { font-weight: bold; }
:last-child Dernier enfant de son parent li:last-child { border: none; } 
/* Exemples d'utilisation */
.bouton {
    background: #FF7A45;
    color: white;
    padding: 10px 20px;
    border: none;
    transition: all 0.3s;
}

.bouton:hover {
    background: #1F2D3D;
    transform: translateY(-2px);
    cursor: pointer;
}

.bouton:active {
    transform: translateY(0);
}

input:focus {
    outline: 2px solid #FF7A45;
    outline-offset: 2px;
}

🎯 Ordre important pour les liens :

Pour les liens, l'ordre des pseudo-classes est important (LOVE HATE) :

a:link { color: blue; }     /* Non visité */
a:visited { color: purple; } 
a:hover { color: red; }      /* Survol */
a:active { color: orange; }  /* Au clic */

🎨 Couleurs et unités

Les différentes façons de définir couleurs et dimensions

🌈 3.1 Les formats de couleurs

CSS propose plusieurs façons de définir les couleurs :

Format Syntaxe Exemple Rendu
Mots-clés Nom de couleur red, blue, green red
Hexadécimal #RRGGBB #FF7A45 #FF7A45
Hexa court #RGB #F00 (rouge) #F00
RGB rgb(r, g, b) rgb(255, 122, 69) rgb(255,122,69)
RGBA rgba(r, g, b, a) rgba(255, 122, 69, 0.5) rgba(255,122,69,0.5)
HSL hsl(h, s%, l%) hsl(17, 100%, 64%) hsl(17,100%,64%)
HSLA hsla(h, s%, l%, a) hsla(17, 100%, 64%, 0.5) hsla(17,100%,64%,0.5)

📖 Explication des formats :

  • Hexadécimal (#RRGGBB) : Chaque paire représente une couleur (Rouge, Vert, Bleu) de 00 à FF. Ex: #FF7A45 = Rouge:255, Vert:122, Bleu:69.
  • RGB : Même principe mais en décimal (0-255). Plus lisible pour certains.
  • RGBA : RGB avec transparence (alpha de 0 à 1).
  • HSL : Teinte (0-360°), Saturation (0-100%), Luminosité (0-100%). Plus intuitif pour créer des variations.

🎯 Exemples d'utilisation :

.exemple1 { color: red; }
.exemple2 { color: #FF7A45; }
.exemple3 { color: rgb(255, 122, 69); }
.exemple4 { color: rgba(255, 122, 69, 0.8); }
.exemple5 { color: hsl(17, 100%, 64%); }
💡
Bon à savoir : Pour la transparence, vous pouvez aussi utiliser opacity: 0.5; qui rend tout l'élément (et ses enfants) transparent, contrairement à rgba/hsla qui ne rend transparent que la couleur elle-même.

📏 3.2 Unités absolues

Les unités absolues ont une taille fixe, indépendante du contexte. Elles sont principalement utilisées pour l'impression.

Unité Nom Équivalence Utilisation
px Pixels 1px = 1/96 de pouce Écrans (très courant)
pt Points 1pt = 1/72 de pouce Impression, typographie
cm Centimètres - Impression
mm Millimètres - Impression
in Pouces 1in = 2.54cm Impression
pc Picas 1pc = 12pt Impression 
/* Exemples d'unités absolues */
.boite {
    width: 200px;
    height: 100px;
    font-size: 16px;
    border: 2px solid black;
}
Pour les écrans : Le pixel (px) est l'unité absolue la plus utilisée. Les autres unités absolues sont rarement utilisées sur le web sauf pour les feuilles de style d'impression.

📐 3.3 Unités relatives

Les unités relatives sont calculées par rapport à une autre valeur (taille de police, dimensions du parent, etc.). Elles sont essentielles pour le responsive design.

Unité Relative à Description
% Parent Pourcentage de la taille du parent
em Police du parent 1em = taille de police du parent (souvent 16px par défaut)
rem Police racine (html) 1rem = taille de police de l'élément html (généralement 16px)
vw Viewport width 1vw = 1% de la largeur de la fenêtre
vh Viewport height 1vh = 1% de la hauteur de la fenêtre
vmin Viewport min 1vmin = 1% de la plus petite dimension (largeur ou hauteur)
vmax Viewport max 1vmax = 1% de la plus grande dimension
ch Caractère "0" Largeur du caractère "0" dans la police courante
ex Hauteur du "x" Hauteur du caractère "x" dans la police courante

Exemples d'utilisation :

/* Unités relatives */
.conteneur {
    width: 80%;          /* 80% de la largeur du parent */
    margin: 0 auto;
}
                          
.texte {
    font-size: 1.2em;      /* 1.2 fois la police du parent */
    margin-bottom: 1.5em;  /* Relatif à la police de l'élément */
}

.rem-exemple {
    font-size: 1.1rem;     /* 1.1 fois la police racine (html) */
    padding: 1rem;
}

.pleine-page {
    width: 100vw;         /* 100% de la largeur de la fenêtre */
    height: 100vh;        /* 100% de la hauteur de la fenêtre */
}

.texte-colonne {
    max-width: 60ch;      /* Largeur idéale pour la lecture (60 caractères) */
}

🎯 Différence entre em et rem :

/* em est cumulatif */
.parent { font-size: 20px; }
.enfant { font-size: 0.8em; } /* 0.8 * 20px = 16px */
.petit-enfant { font-size: 0.8em; } /* 0.8 * 16px = 12.8px */

/* rem est constant */
html { font-size: 16px; }
.parent { font-size: 1.2rem; } /* 1.2 * 16px = 19.2px */
.enfant { font-size: 1.2rem; } /* 1.2 * 16px = 19.2px (identique) */
🎯
Recommandations :
  • Utilisez rem pour les tailles de police (consistant)
  • Utilisez em pour les marges/paddings relatifs à la police locale
  • Utilisez %, vw, vh pour les mises en page responsives
  • Utilisez px pour les bordures, petites valeurs fixes

Exercice 1 : Dégradé de couleurs

Objectif : Créer plusieurs boîtes avec différentes notations de couleurs.

Instructions :

  1. Créez 5 boîtes de 100x100px
  2. Première boîte : couleur hexadécimale (#FF7A45)
  3. Deuxième boîte : couleur RGB (rgb(31, 45, 61))
  4. Troisième boîte : couleur RGBA avec transparence (rgba(255, 122, 69, 0.5))
  5. Quatrième boîte : couleur HSL (hsl(17, 100%, 64%))
  6. Cinquième boîte : couleur avec mot-clé (coral)

Exercice 2 : Mise en page responsive avec unités

Objectif : Utiliser différentes unités pour créer une mise en page.

Instructions :

  1. Créez un conteneur principal qui prend 90% de la largeur et est centré
  2. Ajoutez un header qui prend 100% de la largeur du conteneur

📦 Le modèle de boîte (Box Model)

Margin, padding, border - comprendre comment les éléments occupent l'espace

📐 5.1 Qu'est-ce que le modèle de boîte ?

En CSS, chaque élément est considéré comme une boîte rectangulaire. Le modèle de boîte décrit comment cette boîte est composée.

🔷 Structure d'une boîte (de l'intérieur vers l'extérieur) :

CONTENU
(content)
width: 200px; height: 100px;
  • ① Contenu (content) : Le texte, les images, etc.
  • ② Padding (rembourrage) : Espace entre le contenu et la bordure
  • ③ Border (bordure) : La bordure autour du padding
  • ④ Margin (marge) : Espace entre la bordure et les éléments voisins

📏 5.2 Padding (rembourrage intérieur)

Le padding crée un espace entre le contenu et la bordure. Il fait partie de la boîte et prend la couleur de fond.

/* Padding individuel */
.boite {
    padding-top: 10px;
    padding-right: 15px;
    padding-bottom: 10px;
    padding-left: 15px;
}

/* Padding raccourci - 4 valeurs (haut, droite, bas, gauche) */
.boite {
    padding: 10px 15px 10px 15px;
}

/* Padding - 3 valeurs (haut, gauche/droite, bas) */
.boite {
    padding: 10px 20px 10px;
}

/* Padding - 2 valeurs (haut/bas, gauche/droite) */
.boite {
    padding: 10px 20px;
}

/* Padding - 1 valeur (tous les côtés) */
.boite {
    padding: 10px;
}

🎯 Effet visuel du padding :

padding:0
padding:10px
padding:20px

↔️ 5.3 Margin (marge extérieure)

Le margin crée un espace entre la bordure de l'élément et les éléments voisins. Il est transparent et ne prend pas la couleur de fond.

/* Même syntaxe que padding */
.boite {
    margin: 10px;           /* 10px de tous les côtés */
    margin: 10px 20px;      /* 10px haut/bas, 20px gauche/droite */
    margin: 10px 15px 20px; /* 10px haut, 15px côtés, 20px bas */
}

/* Cas particuliers */
.centre {
    margin: 0 auto;         /* Centre horizontalement un bloc (avec width définie) */
}

.espace-negatif {
    margin-left: -10px;    /* Marge négative (déplace vers la gauche) */
}
⚠️
Fusion des marges (margin collapsing) : Quand deux marges verticales se touchent, elles fusionnent en une seule (la plus grande des deux). C'est normal.
Élément A (margin-bottom: 20px)
Élément B (margin-top: 30px)

L'espace entre eux sera de 30px (pas 50px)

🖼️ 5.4 Border (bordure)

La bordure entoure le padding. Elle peut être stylisée de nombreuses façons.

/* border: largeur style couleur */
.bordure-simple {
    border: 2px solid #1F2D3D;
}

/* Bordures individuelles */
.bordure-haute {
    border-top: 3px dashed #FF7A45;
}

/* Styles de bordure */
.exemples {
    border-style: solid;     /* plein */
    border-style: dashed;    /* tirets */
    border-style: dotted;    /* points */
    border-style: double;    /* double ligne */
    border-style: groove;    /* effet 3D */
    border-style: ridge;     /* effet 3D inverse */
    border-style: inset;     /* enfoncé */
    border-style: outset;    /* relief */
}

/* Border-radius (coins arrondis) */
.arrondi {
    border-radius: 10px;          /* tous les coins */
    border-radius: 10px 20px;    /* haut-gauche/bas-droit, haut-droit/bas-gauche */
    border-radius: 50%;           /* cercle parfait (avec largeur/hauteur égales) */
}

📊 5.5 width et height

Par défaut, width et height s'appliquent au contenu uniquement. Le padding, border et margin s'ajoutent.

.boite-standard {
    width: 200px;
    height: 100px;
    padding: 20px;
    border: 5px solid #1F2D3D;
    /* Largeur totale = 200 + 40 (padding) + 10 (border) = 250px */
}

📖 Calcul de la largeur totale :

Largeur totale = width + padding-left + padding-right + border-left + border-right

Hauteur totale = height + padding-top + padding-bottom + border-top + border-bottom

💡
Solution : box-sizing: border-box

Avec box-sizing: border-box, width et height incluent le padding et la bordure. C'est beaucoup plus intuitif !

* {
    box-sizing: border-box;
}

.boite {
    width: 200px;
    padding: 20px;
    border: 5px solid;
    /* Largeur totale = 200px (inclut padding et border) */
}

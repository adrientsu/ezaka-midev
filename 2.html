🎨 Arrière-plans (background)

Couleurs, images, dégradés et positionnement

🎨 6.1 background-color

La propriété background-color définit la couleur de fond d'un élément.

.boite {
    background-color: #FF7A45;
    background-color: rgb(255, 122, 69);
    background-color: hsl(17, 100%, 64%);
    background-color: transparent;
}

🎯 Exemples :

#FF7A45
#1F2D3D
#6B7280
#F6EDE7

🖼️ 6.2 background-image

Permet de définir une image de fond.

.fond-image {
    background-image: url('images/pattern.png');
    background-image: url('https://example.com/image.jpg');
}

Propriétés associées :

Propriété Description Valeurs
background-repeat Répétition de l'image repeat, repeat-x, repeat-y, no-repeat
background-position Position de l'image center, top, left, bottom, right, 50% 50%
background-size Taille de l'image auto, cover, contain, 100% 100%
background-attachment Fixer l'image scroll, fixed, local 
.hero {
    background-image: url('montagne.jpg');
    background-size: cover;        /* Couvre tout l'élément */
    background-position: center;    /* Centrée */
    background-repeat: no-repeat;   /* Pas de répétition */
    background-attachment: fixed;   /* Fixe pendant le scroll */
}

.pattern {
    background-image: url('pattern.png');
    background-repeat: repeat;       /* Répète l'image */
}

🎯 Différence entre cover et contain :

cover
contain

🌈 6.3 Dégradés (gradients)

CSS permet de créer des dégradés sans images.

Dégradé linéaire

.degrade-lineaire {
    /* direction, couleur1, couleur2 */
    background: linear-gradient(to right, #FF7A45, #1F2D3D);
    
    /* Avec angle */
    background: linear-gradient(45deg, #FF7A45, #1F2D3D, #6B7280);
    
    /* Avec plusieurs couleurs et positions */
    background: linear-gradient(to bottom, #FF7A45 0%, #1F2D3D 50%, #6B7280 100%);
}

Dégradé radial

.degrade-radial {
    background: radial-gradient(circle, #FF7A45, #1F2D3D);
    
    /* Forme et taille */
    background: radial-gradient(circle at top left, #FF7A45, #1F2D3D, #6B7280);
}

📝 6.4 La propriété raccourcie background

background permet de définir toutes les propriétés en une seule ligne.

/* Ordre: color image repeat position/size attachment */
.exemple {
    background: #FF7A45 url('image.jpg') no-repeat center/cover fixed;
}

/* Équivalent à : */
.exemple {
    background-color: #FF7A45;
    background-image: url('image.jpg');
    background-repeat: no-repeat;
    background-position: center;
    background-size: cover;
    background-attachment: fixed;
}
Astuce : On peut combiner plusieurs backgrounds en les séparant par des virgules.

📍 Le positionnement

Static, relative, absolute, fixed, sticky

📌 7.1 position: static (défaut)

C'est le positionnement par défaut. Les éléments suivent le flux normal du document.

.element {
    position: static;  /* Valeur par défaut */
    /* top, right, bottom, left n'ont aucun effet */
}

Caractéristiques :

  • L'élément est positionné selon le flux normal
  • Les propriétés top, right, bottom, left sont ignorées
  • C'est le comportement naturel des éléments

📏 7.2 position: relative

Positionne l'élément par rapport à sa position normale, tout en laissant un espace vide à son emplacement d'origine.

.element {
    position: relative;
    top: 20px;     /* Décalé de 20px vers le bas */
    left: 10px;    /* Décalé de 10px vers la droite */
    /* L'espace d'origine est conservé */
}

🎯 Exemple visuel :

Normal
Relative

L'élément est décalé mais son emplacement d'origine reste vide.

💡
Utilisation courante :
  • Décaler légèrement un élément
  • Servir de point de référence pour un enfant en absolute
  • Créer des effets de superposition légers

🎯 7.3 position: absolute

Positionne l'élément par rapport à son ancêtre positionné le plus proche (sinon par rapport à la fenêtre). L'élément est retiré du flux normal.

.parent {
    position: relative;  /* Point de référence */
    height: 200px;
    border: 1px solid #1F2D3D;
}

.enfant {
    position: absolute;
    top: 20px;
    right: 20px;
    /* Positionné à 20px du haut et 20px de la droite du parent */
}

🎯 Exemple : badge "Nouveau"

Nouveau
Contenu de la carte
.carte {
    position: relative;
}
.badge {
    position: absolute;
    top: -10px;
    right: -10px;
}
Important : Pour qu'un élément en absolute se positionne par rapport à un parent, ce parent doit avoir une position autre que static (souvent relative).

📌 7.4 position: fixed

Positionne l'élément par rapport à la fenêtre du navigateur. Il reste fixe même pendant le défilement.

.header-fixe {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    background: #1F2D3D;
    color: white;
    z-index: 1000;
}

.bouton-retour-haut {
    position: fixed;
    bottom: 20px;
    right: 20px;
}

🎯 Utilisations courantes :

  • Barres de navigation fixes
  • Boutons "Retour en haut"
  • Fenêtres modales (avec overlay)
  • Cookies banners

🧲 7.5 position: sticky

Mélange de relative et fixed : l'élément se comporte comme relative jusqu'à un certain seuil de défilement, puis devient fixe.

.titre-section {
    position: sticky;
    top: 0;  /* Seuil de déclenchement */
    background: #FF7A45;
    color: white;
    padding: 10px;
}

📊 7.6 z-index (superposition)

Contrôle l'ordre de superposition des éléments positionnés. Plus la valeur est élevée, plus l'élément est au-dessus.

.element1 {
    position: absolute;
    z-index: 10;  /* Au-dessus */
}

.element2 {
    position: absolute;
    z-index: 5;   /* En-dessous */
}

📦 Flexbox

Mise en page flexible en une dimension

🔍 8.1 Qu'est-ce que Flexbox ?

Flexbox (Flexible Box) est un mode de mise en page qui permet de répartir l'espace entre des éléments et de les aligner facilement, même quand leurs tailles sont inconnues ou dynamiques.

💡
Avantages de Flexbox :
  • Alignement vertical simple
  • Répartition automatique de l'espace
  • Ordre des éléments indépendant du HTML
  • Adaptation à différentes tailles d'écran

📐 8.2 Le conteneur flex (display: flex)

Pour utiliser Flexbox, on définit d'abord un conteneur avec display: flex.

.conteneur {
    display: flex;           /* Les enfants deviennent des éléments flex */
    /* display: inline-flex; pour un conteneur inline */
}

📊 8.7 align-content (alignement multi-lignes)

Aligne les lignes entre elles quand il y a plusieurs lignes (avec flex-wrap).

.conteneur {
    display: flex;
    flex-wrap: wrap;
    align-content: center;    /* Centre les lignes verticalement */
}

Valeurs : flex-start, flex-end, center, space-between, space-around, stretch

🧩 8.8 Propriétés des éléments enfants

order

Change l'ordre d'affichage des éléments (indépendant du HTML).

.element {
    order: 2;  /* Plus petit nombre = apparaît en premier */
}

flex-grow

Capacité d'un élément à grandir pour remplir l'espace disponible.

.element {
    flex-grow: 1;  /* Prend tout l'espace disponible */
}

.element:nth-child(2) {
    flex-grow: 2;  /* Prend 2 fois plus d'espace que les autres */
}

flex-shrink

Capacité d'un élément à rétrécir si nécessaire.

flex-basis

Taille de base avant distribution de l'espace.

align-self

Permet de surcharger align-items pour un élément spécifique.

.element-special {
    align-self: flex-end;  /* Aligné en bas alors que les autres sont centrés */
}

flex (raccourci)

Combine flex-grow, flex-shrink et flex-basis.

.element {
    flex: 1;               /* flex-grow: 1, flex-shrink: 1, flex-basis: 0 */
    flex: 1 0 200px;       /* grow:1, shrink:0, basis:200px */
}

🔲 CSS Grid

Mise en page en deux dimensions (lignes et colonnes)

🔍 9.1 Qu'est-ce que CSS Grid ?

CSS Grid est un système de mise en page en deux dimensions (lignes ET colonnes). Contrairement à Flexbox qui est unidimensionnel, Grid permet de contrôler à la fois les rangées et les colonnes.

⚖️
Flexbox vs Grid :
  • Flexbox : 1D (ligne OU colonne). Idéal pour les composants, navigation, centrage.
  • Grid : 2D (lignes ET colonnes). Idéal pour la mise en page globale, les grilles complexes.
  • Ils sont complémentaires, pas concurrents !

📐 9.2 Le conteneur grid (display: grid)

Pour utiliser Grid, on définit un conteneur avec display: grid.

.conteneur {
    display: grid;
}

📊 9.3 grid-template-columns

Définit le nombre et la taille des colonnes.

/* 3 colonnes égales */
.conteneur {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
}

/* Colonnes de différentes tailles */
.conteneur {
    grid-template-columns: 200px 1fr 300px;
}

/* Avec repeat() */
.conteneur {
    grid-template-columns: repeat(4, 1fr);  /* 4 colonnes égales */
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* Responsive */
}

📏 9.4 grid-template-rows

Définit la hauteur des lignes.

.conteneur {
    display: grid;
    grid-template-columns: 1fr 1fr;
    grid-template-rows: 100px 200px 150px;
}

🕳️ 9.5 gap, row-gap, column-gap

Définit l'espacement entre les cellules.

.conteneur {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
    gap: 20px;           /* Espacement entre lignes et colonnes */
    row-gap: 10px;        /* Espacement uniquement entre lignes */
    column-gap: 15px;     /* Espacement uniquement entre colonnes */
}

🎯 9.6 Placement des éléments

On peut placer les éléments sur la grille en spécifiant où ils commencent et finissent.

Lignes de grille

Les lignes sont numérotées à partir de 1.

.element {
    grid-column-start: 1;
    grid-column-end: 3;    /* Occupe de la ligne 1 à 3 (2 colonnes) */
    
    /* Raccourci */
    grid-column: 1 / 3;
    
    /* Span (étendre) */
    grid-column: 1 / span 2;  /* Commence à 1 et s'étend sur 2 colonnes */
    
    /* Pour les lignes */
    grid-row: 1 / 3;
    
    /* Raccourci complet : grid-row-start / grid-column-start / grid-row-end / grid-column-end */
    grid-area: 1 / 1 / 3 / 3;
}

📱 Responsive Design

Media queries, mobile-first, points de rupture

🌍 10.1 Qu'est-ce que le Responsive Design ?

Le responsive design est une approche de conception web qui vise à adapter l'affichage d'un site à tous les types d'écrans (mobile, tablette, desktop).

📱
Pourquoi c'est important :
  • Plus de 50% du trafic web est mobile
  • Google favorise les sites responsives (SEO)
  • Une seule codebase à maintenir
  • Meilleure expérience utilisateur

📐 10.2 La balise meta viewport

Essentielle pour le responsive sur mobile, elle contrôle la mise à l'échelle.

<!-- Dans le <head> du HTML -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Explication :

  • width=device-width : utilise la largeur réelle de l'appareil
  • initial-scale=1.0 : pas de zoom initial

📊 10.3 Les media queries

Les media queries permettent d'appliquer des styles conditionnellement selon les caractéristiques de l'appareil.

/* Syntaxe de base */
@media (condition) {
    /* Styles qui s'appliquent quand la condition est vraie */
}

/* Exemples */
@media (max-width: 768px) {
    /* Styles pour écrans de largeur <= 768px (tablette/mobile) */
    body {
        font-size: 14px;
    }
}

@media (min-width: 1024px) {
    /* Styles pour écrans de largeur >= 1024px (desktop) */
    .conteneur {
        max-width: 1200px;
    }
}

/* Combinaisons */
@media (min-width: 768px) and (max-width: 1023px) {
    /* Tablettes */
}

/* Orientation */
@media (orientation: landscape) {
    /* Mode paysage */
}

@media (orientation: portrait) {
    /* Mode portrait */
}

📏 10.4 Points de rupture (breakpoints) standards

Appareil Largeur Media query
Téléphone< 576px@media (max-width: 575px)
Tablette576px - 768px@media (min-width: 576px) and (max-width: 768px)
Desktop769px - 992px@media (min-width: 769px) and (max-width: 992px)
Grand écran> 992px@media (min-width: 993px)

Ou approche plus simple :

/* Mobile : styles de base */
.exemple { width: 100%; }

/* Tablette */
@media (min-width: 768px) {
    .exemple { width: 50%; }
}

/* Desktop */
@media (min-width: 1024px) {
    .exemple { width: 33.33%; }
}

📱💻 10.5 Mobile-first vs Desktop-first

Mobile-first (recommandé)

On part du mobile et on ajoute des styles pour les écrans plus grands.

/* Base : mobile */
.sidebar { display: none; }

/* Tablette et plus */
@media (min-width: 768px) {
    .sidebar { display: block; }
}

✅ Plus simple, plus performant sur mobile

Desktop-first

On part du desktop et on simplifie pour les petits écrans.

/* Base : desktop */
.sidebar { display: block; }

/* Mobile */
@media (max-width: 767px) {
    .sidebar { display: none; }
}

❌ Styles par défaut plus lourds

Recommandation : Utilisez l'approche mobile-first. C'est plus simple et plus performant car les appareils mobiles n'ont pas à charger et annuler des styles desktop.

🖼️ 10.6 Images responsives

Techniques pour adapter les images aux différents écrans.

max-width: 100%

L'image ne dépasse jamais la largeur de son conteneur.

img {
    max-width: 100%;
    height: auto;  /* Maintient les proportions */
}

srcset (plusieurs résolutions)

<img 
    src="image.jpg"
    srcset="image-petite.jpg 300w, image-moyenne.jpg 600w, image-grande.jpg 900w"
    sizes="(max-width: 600px) 300px, (max-width: 1200px) 600px, 900px"
    alt="Description">

🔤 10.7 Typographie responsive

Adapter la taille du texte selon l'écran.

/* Avec media queries */
h1 {
    font-size: 24px;
}

@media (min-width: 768px) {
    h1 { font-size: 32px; }
}

@media (min-width: 1024px) {
    h1 { font-size: 48px; }
}

/* Avec des unités fluides (vw) */
h1 {
    font-size: clamp(24px, 5vw, 48px);  /* Minimum 24px, maximum 48px */
}

🍔 10.8 Menu hamburger (exemple simple)

/* Styles de base (mobile) */
.nav-links {
    display: none;  /* Caché sur mobile */
}

.menu-icon {
    display: block;  /* Affiche l'icône hamburger */
}

/* Desktop */
@media (min-width: 768px) {
    .nav-links {
        display: flex;  /* Affiche le menu */
    }
    
    .menu-icon {
        display: none;  /* Cache l'icône hamburger */
    }
}

✍️ 10.9 Exercices pratiques

Exercice 1 : Grille responsive

Objectif : Créer une grille qui change selon la taille d'écran.

Instructions :

  1. Sur mobile (< 576px) : 1 colonne
  2. Sur tablette (576px - 768px) : 2 colonnes
  3. Sur desktop (> 768px) : 4 colonnes
  4. Ajoutez 8 cartes

Exercice 2 : Navigation responsive

Objectif : Créer une navigation qui s'adapte.

Instructions :

  1. Sur desktop : menu horizontal avec tous les liens
  2. Sur mobile : menu caché, icône hamburger visible
  3. Utilisez l'approche mobile-first

Exercice 3 : Layout complexe responsive

Objectif : Adapter une mise en page Grid selon l'écran.

Instructions :

  1. Sur mobile : tout en colonne (header, main, sidebar, footer)
  2. Sur tablette : header, puis main+sidebar côte à côte, footer
  3. Sur desktop : header, main+sidebar, footer avec sidebar à droite

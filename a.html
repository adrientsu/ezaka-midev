Chapitre 01 30 min Débutant

📌 Introduction au HTML

Comprendre ce qu'est le HTML et comment il fonctionne

🔍 1.1 Qu'est-ce que le HTML ?

HTML signifie HyperText Markup Language (Langage de balisage hypertexte). C'est le langage standard utilisé pour créer les pages web.

📖 Explication approfondie :

Le HTML n'est pas un langage de programmation (comme JavaScript ou Python), mais un langage de balisage. Cela signifie qu'il utilise des "balises" pour décrire la structure et le contenu d'une page web. Pensez-y comme à un squelette : il définit où vont les titres, les paragraphes, les images, etc.

💡
À retenir : Le navigateur lit le code HTML et l'interprète pour afficher une page web structurée. Sans HTML, le web ne serait qu'un amas de texte sans organisation.

📜 Un peu d'histoire :

  • 1991 : Tim Berners-Lee, chercheur au CERN, invente le HTML pour partager des documents scientifiques.
  • 1995 : HTML 2.0 - Première standardisation.
  • 1997 : HTML 3.2 - Introduction des tableaux et des scripts.
  • 1999 : HTML 4.01 - Séparation du contenu et de la présentation (CSS).
  • 2014 : HTML5 - Révolution avec le multimédia natif (vidéo, audio) et les balises sémantiques.

🏗️ 1.2 Structure de base d'une page HTML

Toute page HTML valide suit une structure minimale obligatoire. Voici le squelette de base :

HTML Structure minimale 
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Ma première page</title>
</head>
<body>
    
</body>
</html>

🔬 Analyse ligne par ligne :

<!DOCTYPE html>

Déclare que le document utilise HTML5. C'est obligatoire pour que le navigateur interprète correctement la page.

<html lang="fr">

La balise racine qui englobe tout le contenu. L'attribut lang="fr" indique que la page est en français (utile pour l'accessibilité et le SEO).

<head> ... </head>

Contient les métadonnées (informations sur la page) qui ne sont pas affichées directement. C'est ici qu'on met le titre, les liens CSS, les meta descriptions, etc.

<meta charset="UTF-8">

Définit l'encodage des caractères. UTF-8 permet d'afficher correctement tous les caractères spéciaux (accents, emojis, etc.).

<meta name="viewport" ...>

Essentiel pour le responsive design. Il dit au navigateur comment adapter la page aux différents écrans (mobile, tablette, desktop).

<title>...</title>

Le titre qui apparaît dans l'onglet du navigateur et dans les résultats de recherche.

<body> ... </body>

Contient tout le contenu visible de la page : textes, images, vidéos, etc.

🎯 Exemple concret :

Voici une page HTML minimale avec un peu de contenu :

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Ma première page</title>
</head>
<body>
    <h1>Bonjour le monde !</h1>
    <p>Ceci est mon premier paragraphe en HTML.</p>
</body>
</html>
Rendu dans le navigateur :

Bonjour le monde !

Ceci est mon premier paragraphe en HTML.

🏷️ 1.3 Comprendre les balises

Une balise est un élément de marquage qui indique au navigateur comment afficher le contenu. Les balises sont entourées de chevrons < >.

🔹 Les deux types de balises :

Balises en paires (ouvrante/fermante)

La plupart des balises fonctionnent par paires :

<h1>Titre</h1>
<p>Paragraphe</p>
<div>Conteneur</div>

La balise fermante a un slash / avant le nom.

Balises orphelines (auto-fermantes)

Certaines balises n'ont pas de contenu et se ferment elles-mêmes :

<img src="image.jpg">
<br>
<hr>
<input type="text">

En HTML5, le slash final est optionnel. Les deux sont corrects : <br> ou <br />.

🔹 Les attributs :

Les attributs donnent des informations supplémentaires sur une balise. Ils se placent dans la balise ouvrante.

<img src="photo.jpg" alt="Description" width="300">
<a href="https://google.com" target="_blank">Google</a>

Structure : nom="valeur". Les guillemets sont obligatoires.

📝 Résumé de la leçon 1

  • HTML = HyperText Markup Language (langage de balisage)

  • Les commentaires : <!-- ... --> ne sont pas affichés
Chapitre 02 35 min Débutant

📝 Les balises de texte

Titres, paragraphes et formatage du texte

🔤 2.1 Les titres (h1 à h6)

HTML propose 6 niveaux de titres, de <h1> (le plus important) à <h6> (le moins important).

<h1>Titre principal de la page</h1>
<h2>Sous-titre de section</h2>
<h3>Titre de sous-section</h3>
<h4>Titre niveau 4</h4>
<h5>Titre niveau 5</h5>
<h6>Titre niveau 6</h6>

📖 Règles importantes :

  • Un seul h1 par page : Il représente le titre principal. En avoir plusieurs nuit au SEO.
  • Hiérarchie logique : Ne sautez pas de niveaux (h1 → h3 directement est incorrect).
  • Les titres structurent le contenu : Comme un plan de dissertation.
Rendu visuel :

h1 - Titre principal

h2 - Sous-titre

h3 - Titre niveau 3

h4 - Titre niveau 4

h5 - Titre niveau 5
h6 - Titre niveau 6

📄 2.2 Les paragraphes

La balise <p> définit un paragraphe. Les navigateurs ajoutent automatiquement un espace avant et après.

<p>Ceci est un premier paragraphe. Il peut contenir plusieurs phrases et s'étendre sur plusieurs lignes dans le code source. Le navigateur gère automatiquement le retour à la ligne.</p>

<p>Ceci est un second paragraphe. Il sera séparé du premier par un espace vertical.</p>

🔍 À savoir :

Dans le code source, vous pouvez écrire un paragraphe sur plusieurs lignes. Le navigateur ignorera ces sauts de ligne et formatera le texte selon sa largeur. Pour forcer un retour à la ligne, utilisez <br>.

🎯 Exemple avec retour à la ligne :

<p>
    Première ligne.<br>
    Deuxième ligne après un saut.<br>
    Troisième ligne.
</p>

✨ 2.3 Les balises de formatage

HTML offre plusieurs balises pour donner du style et du sens au texte :

Balise Description Exemple Rendu
<strong> Texte important (gras) <strong>Attention !</strong> Attention !
<em> Emphase (italique) <em>Très important</em> Très important
<mark> Texte surligné <mark>À retenir</mark> À retenir
<small> Texte plus petit <small>Mentions légales</small> Mentions légales
<del> Texte supprimé (barré) <del>Ancien prix 100€</del> Ancien prix 100€
<ins> Texte inséré (souligné) <ins>Nouveau prix 80€</ins> Nouveau prix 80€
<sub> Indice H<sub>2</sub>O H2O
<sup> Exposant 10<sup>2</sup> = 100 102 = 100
<blockquote> Citation longue <blockquote>Citation...</blockquote> Citation en retrait
<q> Citation courte <q>Bonjour</q> Bonjour
<abbr> Abréviation <abbr title="HyperText Markup Language">HTML</abbr> HTML
<code> Code informatique <code>var x = 10;</code> var x = 10;
<pre> Texte préformaté <pre> Ligne1 Ligne2</pre> Conserve les espaces

🎯 Exemple combiné :

<p>
    <strong>Important :</strong> 
    Le <em>HTML<em> est un langage de <mark>balisage</mark>.
    La molécule d'eau est H<sub>2</sub>O.
    En maths, 10<sup>2</sup> = 100.
    <small>(c) 2026</small>
</p>
Rendu :

Important : Le HTML est un langage de balisage. La molécule d'eau est H2O. En maths, 102 = 100. (c) 2026

Chapitre 03 25 min Débutant

📋 Les listes

Listes ordonnées, non-ordonnées et de définition

🔹 3.1 Les listes non-ordonnées (ul)

La balise <ul> (Unordered List) crée une liste avec des puces. Chaque élément est dans <li> (List Item).

<ul>
    <li>HTML</li>
    <li>CSS</li>
    <li>JavaScript</li>
</ul>
Rendu :
  • HTML
  • CSS
  • JavaScript

🎨 Personnalisation avec CSS :

<!-- Changer le style des puces -->
<ul style="list-style-type: square;">...</ul>
<ul style="list-style-type: circle;">...</ul>
<ul style="list-style-type: none;">...</ul>

🔢 3.2 Les listes ordonnées (ol)

La balise <ol> (Ordered List) crée une liste numérotée.

<ol>
    <li>Préparer le matériel</li>
    <li>Écrire le code</li>
    <li>Tester la page</li>
    <li>Mettre en ligne</li>
</ol>
Rendu :
  1. Préparer le matériel
  2. Écrire le code
  3. Tester la page
  4. Mettre en ligne

⚙️ Attributs des listes ordonnées :

Attribut Description Exemple
type Type de numérotation type="1" (1,2,3)
type="A" (A,B,C)
type="a" (a,b,c)
type="I" (I,II,III)
type="i" (i,ii,iii)
start Valeur de départ start="5" commence à 5
reversed Ordre inverse reversed 
<ol type="A" start="3">
    <li>Troisième lettre (C)</li>
    <li>Quatrième lettre (D)</li>
</ol>

🪆 3.3 Les listes imbriquées

On peut mettre des listes dans des listes pour créer des hiérarchies.

<ul>
    <li>Langages front-end
        <ul>
            <li>HTML</li>
            <li>CSS</li>
            <li>JavaScript</li>
        </ul>
    </li>
    <li>Langages back-end
        <ul>
            <li>PHP</li>
            <li>Python</li>
            <li>Java</li>
        </ul>
    </li>
</ul>
Rendu :
  • Langages front-end
    • HTML
    • CSS
    • JavaScript
  • Langages back-end
    • PHP
    • Python
    • Java

📚 3.4 Les listes de définition (dl, dt, dd)

Les listes de définition servent à présenter des termes et leurs définitions.

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language - Langage de balisage pour le web</dd>
    
    <dt>CSS</dt>
    <dd>Cascading Style Sheets - Langage de style pour le web</dd>
    
    <dt>JS</dt>
    <dd>JavaScript - Langage de programmation pour le web</dd>
</dl>
Rendu :
HTML
HyperText Markup Language - Langage de balisage pour le web
CSS
Cascading Style Sheets - Langage de style pour le web

Chapitre 04 30 min Débutant

🔗 Les liens

Ancres, chemins absolus/relatifs, attributs

🔤 4.1 Syntaxe de base

La balise <a> (anchor) crée un lien hypertexte. L'attribut href définit la destination.

<a href="https://google.com">Aller sur Google</a>

📖 Structure :

Le texte entre les balises est le texte du lien (ce que voit l'utilisateur). L'attribut href est la cible (où on va).

📍 4.2 Chemins absolus vs relatifs

Chemin absolu

URL complète avec protocole (http:// ou https://)

<a href="https://google.com">Google</a>
<a href="https://example.com/page.html">Page externe</a>

Utilisé pour les liens vers d'autres sites web.

Chemin relatif

Chemin par rapport à la page actuelle

<a href="page2.html">Page dans même dossier</a>
<a href="dossier/page.html">Dans sous-dossier</a>
<a href="../page.html">Dossier parent</a>

Utilisé pour naviguer dans votre propre site.

📁 Explication des chemins :

Chemin Signification
fichier.htmlFichier dans le même dossier
dossier/fichier.htmlFichier dans un sous-dossier
../fichier.htmlFichier dans le dossier parent
../../fichier.htmlFichier deux niveaux au-dessus
/fichier.htmlÀ partir de la racine du site

⚙️ 4.3 Attributs importants

Attribut Description Exemple
target Où ouvrir le lien target="_blank" (nouvel onglet)
target="_self" (même onglet - défaut)
title Infobulle au survol title="Visiter Google"
download Forcer le téléchargement download="fichier.pdf"
rel Relation avec le lien rel="noopener" (sécurité) 
<a href="https://google.com" 
   target="_blank" 
   title="Ouvrir Google dans un nouvel onglet"
   rel="noopener">
    Google
</a>

⚓ 4.4 Les ancres (liens internes)

Les ancres permettent de naviguer à l'intérieur d'une même page.

<!-- 1. Créer l'ancre (destination) -->
<h2 id="section1">Section 1</h2>

<!-- 2. Lien vers l'ancre -->
<a href="#section1">Aller à la section 1</a>

<!-- 3. Lien vers ancre d'une autre page -->
<a href="page.html#section2">Aller à section 2 sur autre page</a>

🎯 Exemple concret :

<!-- Menu de navigation -->
<nav>
    <a href="#presentation">Présentation</a>
    <a href="#competences">Compétences</a>
    <a href="#contact">Contact</a>
</nav>

<!-- Sections correspondantes -->
<section id="presentation">...</section>
<section id="competences">...</section>
<section id="contact">...</section>

Chapitre 05 30 min Débutant

🖼️ Les images

Formats, attributs, figures, responsive

🖼️ 5.1 La balise <img>

La balise <img> est une balise orpheline qui affiche des images.

<img src="chemin/vers/image.jpg" alt="Description de l'image">

📖 Attributs obligatoires :

  • src (source) : chemin vers le fichier image
  • alt (alternative) : texte décrivant l'image (pour accessibilité et si l'image ne charge pas)

🎯 Exemple :

<img src="images/logo.png" 
     alt="Logo de notre site"
     width="200"
     height="100">

📁 5.2 Les formats d'images web

Format Description Utilisation
JPEG/JPG Format avec perte, bonne compression Photos, images complexes
PNG Sans perte, supporte la transparence Logos, icônes, captures d'écran
GIF Animations, 256 couleurs max Animations simples
SVG Vectoriel (code XML), redimensionnable Logos, icônes, illustrations
WebP Format moderne, meilleure compression Tous types (support moderne)
AVIF Très bonne compression, support limité Images modernes

⚙️ 5.3 Attributs des images

Attribut Description Exemple
width Largeur en pixels width="300"
height Hauteur en pixels height="200"
loading Chargement paresseux loading="lazy"
title Infobulle au survol title="Notre équipe"
srcset Plusieurs résolutions (responsive) srcset="petit.jpg 300w, grand.jpg 600w" 
<!-- Image responsive avec srcset -->
<img 
    src="image.jpg"
    srcset="image-petite.jpg 300w, image-moyenne.jpg 600w, image-grande.jpg 900w"
    sizes="(max-width: 600px) 300px, (max-width: 1200px) 600px, 900px"
    alt="Description"
    loading="lazy">

📸 5.4 Figures et légendes

Les balises <figure> et <figcaption> associent une image à sa légende.

<figure>
    <img src="graphique.png" alt="Graphique des ventes">
    <figcaption>Figure 1: Évolution des ventes en 2026</figcaption>
</figure>
Rendu :

🔗 5.5 Images cliquables

Une image peut être un lien en l'entourant d'une balise <a>.

<a href="https://example.com">
    <img src="bouton.png" alt="Cliquez ici">
</a>

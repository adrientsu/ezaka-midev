📝 Les formulaires PHP

Récupérer les données envoyées par l'utilisateur

🔍 11.1 Formulaire HTML et PHP

Les formulaires permettent à l'utilisateur d'envoyer des données au serveur. PHP récupère ces données via les superglobales $_GET et $_POST.

💡
Différence GET/POST :
  • GET : données dans l'URL (visible, limité à 2048 caractères) - pour recherches, filtres
  • POST : données dans le corps de la requête (caché, pas de limite) - pour formulaires sensibles

🔍 11.2 Méthode GET

HTML + PHP Formulaire GET 

<form action="traitement_get.php" method="GET">
    <label for="nom">Nom :</label>
    <input type="text" name="nom" id="nom">
    
    <label for="age">Âge :</label>
    <input type="number" name="age" id="age">
    
    <button type="submit">Envoyer</button>
</form>

URL générée : traitement_get.php?nom=Jean&age=25

🔐 11.3 Méthode POST


<form action="traitement_post.php" method="POST">
    <label for="email">Email :</label>
    <input type="email" name="email" id="email">
    
    <label for="password">Mot de passe :</label>
    <input type="password" name="password" id="password">
    
    <button type="submit">Se connecter</button>
</form>
Astuce : Utilisez $_SERVER['REQUEST_METHOD'] pour vérifier si le formulaire a été soumis en POST, et l'opérateur ?? pour éviter les erreurs si une clé n'existe pas.

📦 11.4 $_REQUEST (les deux)

$_REQUEST contient les données de GET, POST et COOKIE. À utiliser avec précaution.

🔐 Les sessions

Maintenir l'état entre les pages

🔍 14.1 Qu'est-ce qu'une session ?

Une session permet de stocker des données sur le serveur associées à un visiteur unique. Cela permet de suivre un utilisateur à travers les pages.

Contrairement aux cookies, les données de session sont stockées sur le serveur. Le client reçoit seulement un identifiant (PHPSESSID).

🚀 14.2 Démarrer une session

⚠️
Important : session_start() doit être appelé avant tout affichage HTML, avant les cookies, avant tout echo.

🍪 Les cookies

Stocker des données côté client

🔍 15.1 Qu'est-ce qu'un cookie ?

Un cookie est un petit fichier texte stocké côté client (navigateur). Il permet de conserver des informations entre les visites.

Contrairement aux sessions, les cookies sont persistants (si on leur donne une date d'expiration) et stockés sur l'ordinateur de l'utilisateur.

➕ 15.2 setcookie()

⚠️
Important : setcookie() doit être appelé avant tout affichage HTML (comme session_start()).

Critère Session Cookie
Stockage Côté serveur Côté client
Sécurité Plus sécurisé Moins sécurisé (visible par l'utilisateur)
Durée de vie Jusqu'à fermeture navigateur (par défaut) Peut être persistante
Taille max Illimité (selon serveur) 4 Ko par cookie
Données sensibles ✅ Oui ❌ Non
Préférences utilisateur ❌ Disparaît à la fermeture ✅ Persiste

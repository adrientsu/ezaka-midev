Chapitre 11 30 min Avancé

🏷️ Les attributs globaux

Attributs universels applicables à toutes les balises HTML

🔍 11.1 Qu'est-ce qu'un attribut global ?

Les attributs globaux sont des attributs qui peuvent être utilisés sur tous les éléments HTML, sans exception. Ils fournissent des fonctionnalités communes à toutes les balises.

🌐
Liste complète des attributs globaux : accesskey, autocapitalize, class, contenteditable, contextmenu, data-*, dir, draggable, enterkeyhint, hidden, id, inputmode, is, itemid, itemprop, itemref, itemscope, itemtype, lang, nonce, part, popover, slot, spellcheck, style, tabindex, title, translate

🆔 11.2 id et class (les plus utilisés)

id (identifiant unique)

Règle : Un id doit être unique dans toute la page.

<div id="header">En-tête</div>
<div id="menu">Menu</div>
<div id="footer">Pied de page</div>

Utilisations : ancres, JavaScript, CSS spécifique.

class (classe)

Peut être partagée par plusieurs éléments.

<div class="produit">Produit 1</div>
<div class="produit promotion">Produit 2</div>
<div class="produit">Produit 3</div>

Plusieurs classes possibles (séparées par espaces).

📖 Règles de nommage :

  • Commencer par une lettre (a-z A-Z)
  • Peut contenir lettres, chiffres, tirets (-), underscores (_)
  • Pas d'espaces (pour class, plusieurs classes sont séparées par espaces)
  • Sensible à la casse (id="Menu" ≠ id="menu")

📊 11.3 Attributs data-* (données personnalisées)

Les attributs data-* permettent de stocker des informations personnalisées dans les balises HTML, accessibles via JavaScript.

<div 
    data-id="123"
    data-nom="Jean"
    data-prix="25.99"
    data-en-stock="true"
    data-couleurs='["rouge","bleu","vert"]'>
    Informations produit
</div>

🎯 Exemple concret avec JavaScript :

<!-- HTML -->
<button 
    data-produit-id="456"
    data-categorie="electronique"
    onclick="acheter(this)">
    Acheter
</button>

<!-- JavaScript -->
<script>
function acheter(element) {
    const id = element.dataset.produitId;
    const categorie = element.dataset.categorie;
    console.log(`Achat du produit ${id} (catégorie: ${categorie})`);
}
</script>

En JavaScript, on accède aux data-* via element.dataset (ex: dataset.produitId)

Bon à savoir : Les noms d'attributs data-* sont automatiquement convertis en camelCase en JavaScript : data-user-id → dataset.userId

✏️ 11.4 contenteditable (contenu éditable)

L'attribut contenteditable rend le contenu d'un élément modifiable par l'utilisateur.

<div contenteditable="true">
    Ce texte peut être modifié par l'utilisateur.
</div>

<h1 contenteditable="true">Titre modifiable</h1>

<ul contenteditable="true">
    <li>Élément 1</li>
    <li>Élément 2</li>
</ul>

🎯 Rendu :

Cliquez ici et modifiez ce texte !

Valeurs possibles :

  • contenteditable="true" : élément éditable
  • contenteditable="false" : non éditable
  • contenteditable="plaintext-only" : éditable mais texte seul (pas de formatage)

👻 11.5 hidden (élément caché)

L'attribut hidden (booléen) masque un élément de l'affichage.

<p hidden>Ce paragraphe est caché</p>

<!-- Peut être combiné avec JavaScript pour afficher/cacher -->
<div id="message" hidden>Message important</div>
<button onclick="document.getElementById('message').hidden = false">
    Afficher le message
</button>

Note : hidden est l'équivalent de display: none en CSS.

↹ 11.6 tabindex (ordre de tabulation)

tabindex contrôle l'ordre de navigation au clavier (touche Tab).

Valeur Comportement
tabindex="0" Élément focusable, dans l'ordre naturel du document
tabindex="-1" Élément focusable uniquement par JavaScript (pas par Tab)
tabindex="1" (positif) Ordre personnalisé (1 passe en premier, puis 2, etc.) 
<input type="text" tabindex="2" placeholder="Deuxième">
<input type="text" tabindex="1" placeholder="Premier">
<input type="text" tabindex="3" placeholder="Troisième">
<div tabindex="0">Élément focusable (div)</div>
⚠️
Accessibilité : Évitez d'utiliser des valeurs positives (sauf cas très spécifiques). Privilégiez l'ordre naturel du document.

🖱️ 11.7 draggable (élément déplaçable)

Rend un élément déplaçable par glisser-déposer.

<div draggable="true" ondragstart="event.dataTransfer.setData('text/plain', 'valeur')">
    Élément déplaçable
</div>

Valeurs : true (déplaçable) ou false (non déplaçable).

🌐 11.8 lang et dir (langue et direction)

lang (langue)

<p lang="fr">Texte en français</p>
<p lang="en">English text</p>
<p lang="mg">Lahatsoratra malagasy</p>

Important pour l'accessibilité (prononciation) et le SEO.

dir (direction)

<p dir="ltr">Gauche à droite (français)</p>
<p dir="rtl">يمين إلى يسار (arabe)</p>
<p dir="auto">Détection automatique</p>

ltr (left-to-right), rtl (right-to-left), auto.

💬 11.9 title (infobulle)

Affiche une infobulle au survol de l'élément.

<abbr title="HyperText Markup Language">HTML</abbr>
<button title="Cliquez pour enregistrer">💾</button>
<a href="#" title="Lien vers la page d'accueil">Accueil</a>

Survolez ce texte pour voir l'infobulle.

🔍 SEO et accessibilité

Optimiser son site pour les moteurs de recherche et tous les utilisateurs

📈 12.1 Pourquoi SEO et accessibilité ?

SEO (Search Engine Optimization)

Optimisation pour les moteurs de recherche (Google, Bing...)

  • ✅ Meilleur classement dans les résultats
  • ✅ Plus de visiteurs
  • ✅ Crédibilité et confiance

Accessibilité (a11y)

Rendre le site utilisable par tous, y compris personnes handicapées

  • ✅ Non-voyants (lecteurs d'écran)
  • ✅ Personnes âgées
  • ✅ Mobilité réduite
  • ✅ Obligation légale dans certains pays
🎯
Bon à savoir : Les bonnes pratiques d'accessibilité améliorent souvent le SEO !

🏷️ 12.2 Meta tags essentiels pour le SEO

Les meta tags se placent dans la section <head>.

Meta tag Description Exemple
title Titre de la page (affiché dans les résultats) <title>Tutoriel HTML complet</title>
meta description Description courte (150-160 caractères) <meta name="description" content="Apprenez HTML de A à Z...">
meta keywords Mots-clés (moins important aujourd'hui) <meta name="keywords" content="html, cours, tutoriel">
meta author Auteur de la page <meta name="author" content="Ambanivolo mi-DEV">
meta robots Instructions pour les robots <meta name="robots" content="index, follow">
link canonical URL principale (évite le contenu dupliqué) <link rel="canonical" href="https://site.com/page">
HTML Exemple complet 
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- SEO de base -->
    <title>Tutoriel HTML Complet - Apprenez HTML en ligne</title>
    <meta name="description" content="Cours HTML gratuit pour débutants et avancés. 20 leçons détaillées avec exercices pratiques. Maîtrisez le langage du web.">
    <meta name="keywords" content="HTML, cours HTML, tutoriel HTML, apprendre HTML, balises HTML">
    <meta name="author" content="Ambanivolo mi-DEV">
    <meta name="robots" content="index, follow">
    
    <!-- URL canonique -->
    <link rel="canonical" href="https://ambanivolo-dev.mg/html">
</head>

♿ 12.4 ARIA (Accessible Rich Internet Applications)

ARIA est un ensemble d'attributs pour améliorer l'accessibilité des applications web complexes.

♿
Règle d'or : Utilisez les éléments HTML sémantiques natifs quand c'est possible. ARIA est un complément, pas un remplacement.

Rôles ARIA

Définissent le type d'élément pour les lecteurs d'écran.

Rôle Description Exemple
role="navigation"Menu de navigation<div role="navigation">...</div>
role="main"Contenu principal<div role="main">...</div>
role="banner"En-tête<header role="banner">...</header>
role="contentinfo"Pied de page<footer role="contentinfo">...</footer>
role="search"Barre de recherche<div role="search">...</div>
role="form"Formulaire<div role="form">...</div>
role="button"Bouton<div role="button" tabindex="0">Cliquez</div>
role="alert"Message d'alerte<div role="alert">Erreur</div>
role="dialog"Fenêtre modale<div role="dialog">...</div>

Propriétés ARIA

Donnent des informations supplémentaires sur l'état des éléments.

<!-- aria-label : étiquette alternative -->
<button aria-label="Fermer la fenêtre">X</button>

<!-- aria-labelledby : lié à un autre élément -->
<h2 id="titre">Formulaire d'inscription</h2>
<div aria-labelledby="titre">...</div>

<!-- aria-describedby : description supplémentaire -->
<input type="text" aria-describedby="aide-nom">
<div id="aide-nom">Entrez votre nom complet</div>

<!-- aria-required : champ obligatoire -->
<input type="text" aria-required="true">

<!-- aria-disabled : désactivé -->
<button aria-disabled="true">Envoyer</button>

<!-- aria-expanded : état déplié/replié -->
<button aria-expanded="false">Afficher plus</button>

<!-- aria-hidden : cacher des lecteurs d'écran -->
<i class="fas fa-search" aria-hidden="true"></i>

🎯 Exemple complet avec ARIA :

<nav role="navigation" aria-label="Menu principal">
    <ul>
        <li><a href="/" aria-current="page">Accueil</a></li>
        <li><a href="/cours">Cours</a></li>
    </ul>
</nav>

<main role="main">
    <h1>Bienvenue</h1>
    
    <button 
        aria-expanded="false"
        aria-controls="details">
        Voir détails
    </button>
    
    <div id="details" hidden>
        Détails supplémentaires...
    </div>
</main>

🔤 13.1 Qu'est-ce qu'une entité HTML ?

Les entités HTML sont des codes spéciaux qui représentent des caractères qui ont une signification particulière en HTML ou qui ne sont pas disponibles sur le clavier.

Syntaxe : &nom; (nommée) ou &#code; (numérique)

⚠️
Pourquoi utiliser des entités ? Certains caractères comme < et > sont interprétés comme des balises par le navigateur. Les entités permettent de les afficher normalement.

🔒 13.3 L'espace insécable (&nbsp;)

L'espace insécable empêche un retour à la ligne entre deux mots.

Sans espace insécable :

M. Dupont

Peut couper :
M.
Dupont

Avec espace insécable :

M.&nbsp;Dupont

Restera ensemble :
M. Dupont

Cas d'usage :

  • Initiales (M. Dupont)
  • Nombres (10 000 €)
  • Unités (10 km)
  • Prénoms composés (Jean-Pierre)
  • Éviter les veuves orphelines en typographie

🖼️ Iframes avancés

Sécurité, sandbox, communication, intégrations

📌 14.1 Rappel : balise <iframe>

<iframe> (inline frame) permet d'intégrer une autre page HTML dans la page courante.

<iframe 
    src="https://example.com"
    width="100%"
    height="400"
    title="Description"
    frameborder="0"
    loading="lazy"
    allowfullscreen>
    <!-- Message si iframe non supporté -->
    <p>Votre navigateur ne supporte pas les iframes.</p>
</iframe>

🔒 14.2 Attributs de sécurité

sandbox

L'attribut sandbox applique des restrictions de sécurité à l'iframe.

Valeur Description
allow-formsAutorise la soumission de formulaires
allow-modalsAutorise les fenêtres modales (alert, confirm)
allow-orientation-lockAutorise le verrouillage de l'orientation
allow-pointer-lockAutorise le verrouillage du pointeur
allow-popupsAutorise les popups
allow-popups-to-escape-sandboxAutorise les popups à échapper au sandbox
allow-presentationAutorise le mode présentation
allow-same-originAutorise à traiter le contenu comme de la même origine
allow-scriptsAutorise l'exécution de scripts
allow-top-navigationAutorise la navigation dans le contexte parent
allow-top-navigation-by-user-activationNavigation top seulement après action utilisateur
 
<!-- Iframe très restrictive (rien autorisé) -->
<iframe src="page.html" sandbox></iframe>

<!-- Iframe avec scripts et formulaires autorisés -->
<iframe 
    src="page.html" 
    sandbox="allow-scripts allow-forms">
</iframe>

<!-- Iframe de confiance (même origine) -->
<iframe 
    src="https://mon-site.com/page.html" 
    sandbox="allow-same-origin allow-scripts allow-forms">
</iframe>

Autres attributs de sécurité

<!-- referrerpolicy : contrôle les infos envoyées au référent -->
<iframe 
    src="https://example.com"
    referrerpolicy="no-referrer">
</iframe>

<!-- allow : contrôle des fonctionnalités (geolocation, camera, etc.) -->
<iframe 
    src="https://example.com"
    allow="geolocation 'self'; microphone 'none'">
</iframe>
⚠️
Sécurité : Toujours utiliser sandbox pour les iframes de sources non fiables. N'autoriser que ce qui est strictement nécessaire.

📡 14.3 Communication parent-enfant (postMessage)

La méthode postMessage permet une communication sécurisée entre la page parente et l'iframe.

Page parente

<!-- HTML -->
<iframe id="monIframe" 
    src="enfant.html">
</iframe>

<!-- JavaScript -->
<script>
    const iframe = document.getElementById('monIframe');
    
    // Envoyer un message à l'iframe
    iframe.contentWindow.postMessage({
        type: 'INFO',
        data: 'Message du parent'
    }, 'https://enfant.example.com');
    
    // Recevoir des messages de l'iframe
    window.addEventListener('message', (event) => {
        if (event.origin !== 'https://enfant.example.com') return;
        console.log('Message reçu:', event.data);
    });
</script>

Page enfant (iframe)

<!-- JavaScript dans enfant.html -->
<script>
    // Recevoir des messages du parent
    window.addEventListener('message', (event) => {
        if (event.origin !== 'https://parent.example.com') return;
        console.log('Message du parent:', event.data);
        
        // Répondre au parent
        event.source.postMessage({
            type: 'REPONSE',
            data: 'Message de l\'enfant'
        }, event.origin);
    });
</script>

🎯 Exemple d'utilisation :

  • Widgets : communication entre un widget et la page
  • Jeux : score, actions
  • Formulaires : validation, soumission
  • Authentification : login via iframe

⚡ 14.4 Optimisation des performances

loading="lazy"

Charge l'iframe seulement quand il est visible.

<iframe src="https://example.com" loading="lazy"></iframe>

srcdoc (contenu HTML direct)

Permet d'injecter du HTML directement sans fichier externe.

<iframe 
    srcdoc="<p>Contenu HTML directement dans l'iframe</p>"
    width="100%"
    height="100">
</iframe>

name (ciblage)

Permet de cibler l'iframe depuis des liens.

<iframe name="cadre" src="page1.html"></iframe>
<a href="page2.html" target="cadre">Charger page2 dans l'iframe</a>

📊 Microdonnées et Schema.org

Structurer les données pour les moteurs de recherche

🔍 15.1 Qu'est-ce que les microdonnées ?

Les microdonnées sont une façon d'ajouter des informations structurées dans une page HTML pour aider les moteurs de recherche à comprendre le contenu.

🎯
Résultat : Les microdonnées permettent d'obtenir des rich snippets (résultats enrichis) dans Google : étoiles d'avis, prix, événements, etc.

Vocabulaire standard : Schema.org (projet commun de Google, Microsoft, Yahoo, Yandex)

📝 15.2 Syntaxe des microdonnées

Attribut Description Exemple
itemscope Déclare un élément contenant des microdonnées <div itemscope>
itemtype Type de l'élément (URL Schema.org) itemtype="https://schema.org/Product"
itemprop Propriété de l'élément itemprop="name"
itemid Identifiant unique global itemid="sku:12345"
itemref Référence à d'autres éléments itemref="adresse" 
<div itemscope itemtype="https://schema.org/Product">
    <h1 itemprop="name">Smartphone XYZ</h1>
    <img itemprop="image" src="smartphone.jpg" alt="Smartphone">
    <p itemprop="description">Dernier modèle avec écran HD</p>
    
    <div itemprop="offers" itemscope itemtype="https://schema.org/Offer">
        <span itemprop="price">299</span>
        <span itemprop="priceCurrency">EUR</span>
        <link itemprop="availability" href="https://schema.org/InStock">En stock
    </div>
</div>

📋 15.3 Types Schema.org courants

Type Description Propriétés courantes
Product Produit name, description, image, offers, brand, review
Offer Offre (prix, disponibilité) price, priceCurrency, availability, seller
Review Avis reviewRating, author, reviewBody
AggregateRating Note moyenne ratingValue, reviewCount, bestRating
Person Personne name, email, telephone, address, jobTitle
Organization Organisation name, url, logo, address, email
Event Événement name, startDate, endDate, location, description
Place Lieu name, address, geo
LocalBusiness Commerce local name, address, telephone, openingHours
Recipe Recette name, recipeIngredient, recipeInstructions, cookTime
Article Article headline, author, datePublished, articleBody
BreadcrumbList Fil d'Ariane itemListElement (avec ListItem)

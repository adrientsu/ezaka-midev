📊 Les tableaux avancés

Fusion de cellules, en-têtes complexes, structuration

🔍 6.1 Rappel : Structure de base

Un tableau HTML se construit avec les balises suivantes :

  • <table> : conteneur principal
  • <tr> : ligne (table row)
  • <td> : cellule (table data)
  • <th> : cellule d'en-tête (table header)
HTML Tableau simple 
<table>
    <tr>
        <th>Nom</th>
        <th>Âge</th>
        <th>Ville</th>
    </tr>
    <tr>
        <td>Jean</td>
        <td>25</td>
        <td>Paris</td>
    </tr>
</table>

🔄 6.2 Fusion de cellules (colspan et rowspan)

Les attributs colspan et rowspan permettent de fusionner des cellules.

colspan (fusion horizontale)

Fusionne plusieurs colonnes

<td colspan="2">Texte sur 2 colonnes</td>

rowspan (fusion verticale)

Fusionne plusieurs lignes

<td rowspan="3">Texte sur 3 lignes</td>

📋 Exemple d'emploi du temps

<table border="1">
    <tr>
        <th>Heure</th>
        <th>Lundi</th>
        <th>Mardi</th>
        <th>Mercredi</th>
    </tr>
    <tr>
        <td>8h - 10h</td>
        <td>HTML</td>
        <td colspan="2">Projet</td>
    </tr>
    <tr>
        <td>10h - 12h</td>
        <td rowspan="2">CSS</td>
        <td>JavaScript</td>
        <td>PHP</td>
    </tr>
    <tr>
        <td>14h - 16h</td>
        <td>Base de données</td>
        <td>Révision</td>
    </tr>
</table>
Rendu visuel :
Heure Lundi Mardi Mercredi
8h - 10h HTML Projet
10h - 12h CSS JavaScript PHP
14h - 16h Base de données Révision

📖 Règles importantes :

  • colspan="n" : la cellule occupe n colonnes (les cellules suivantes sont décalées)
  • rowspan="n" : la cellule occupe n lignes (il faut sauter les cellules dans les lignes suivantes)
  • Le nombre total de colonnes doit être cohérent sur toutes les lignes

🏷️ 6.3 En-têtes de tableau (thead, tbody, tfoot)

HTML5 permet de structurer un tableau en trois parties :

  • <thead> : en-tête du tableau (une seule fois)
  • <tbody> : corps du tableau (peut être multiple)
  • <tfoot> : pied du tableau (une seule fois)
<table>
    <caption>Ventes trimestrielles 2026</caption>
    
    <thead>
        <tr>
            <th>Produit</th>
            <th>T1</th>
            <th>T2</th>
            <th>T3</th>
            <th>T4</th>
        </tr>
    </thead>
    
    <tbody>
        <tr>
            <th>Ordinateurs</th>
            <td>150</td>
            <td>180</td>
            <td>200</td>
            <td>250</td>
        </tr>
        <tr>
            <th>Téléphones</th>
            <td>300</td>
            <td>320</td>
            <td>280</td>
            <td>400</td>
        </tr>
    </tbody>
    
    <tfoot>
        <tr>
            <th>Total</th>
            <td>450</td>
            <td>500</td>
            <td>480</td>
            <td>650</td>
        </tr>
    </tfoot>
</table>

🔍 Pourquoi utiliser thead/tbody/tfoot ?

  • Accessibilité : les lecteurs d'écran comprennent mieux la structure
  • Impression : l'en-tête peut être répété sur chaque page
  • Défilement : l'en-tête peut rester fixe pendant le scroll
  • Style : possibilité de styliser différemment chaque section

🏗️ 6.4 Tableaux complexes avec en-têtes multiples

On peut créer des tableaux avec des en-têtes sur plusieurs niveaux.

<table>
    <thead>
        <!-- Première ligne d'en-tête -->
        <tr>
            <th rowspan="2">Région</th>
            <th colspan="2">2025</th>
            <th colspan="2">2026</th>
        </tr>
        
        <!-- Deuxième ligne d'en-tête -->
        <tr>
            <th>Ventes</th>
            <th>Objectif</th>
            <th>Ventes</th>
            <th>Objectif</th>
        </tr>
    </thead>
    
    <tbody>
        <tr>
            <th>Nord</th>
            <td>100</td>
            <td>110</td>
            <td>120</td>
            <td>130</td>
        </tr>
        <tr>
            <th>Sud</th>
            <td>80</td>
            <td>90</td>
            <td>100</td>
            <td>110</td>
        </tr>
    </tbody>
</table>
Astuce : Pour des tableaux complexes, dessinez d'abord la structure sur papier pour visualiser les fusions.

📝 Les formulaires (partie 1)

Structure, inputs, labels, boutons

🏗️ 7.1 La balise <form>

La balise <form> est le conteneur de tous les éléments d'un formulaire.

<form action="traitement.php" method="post">
    <!-- Tous les champs du formulaire -->
    <button type="submit">Envoyer</button>
</form>

Attributs importants de <form> :

Attribut Description Valeurs
action URL où les données sont envoyées URL absolue ou relative
method Méthode HTTP d'envoi get (visible dans l'URL) ou post (caché)
name Nom du formulaire texte
enctype Encodage des données multipart/form-data pour fichiers
target Où afficher la réponse _blank, _self, etc.

📖 GET vs POST :

  • GET : les données sont dans l'URL (ex: ?nom=Jean&age=25). Limité à 2048 caractères. Pour les recherches, filtres.
  • POST : les données sont dans le corps de la requête. Pas de limite. Pour les mots de passe, fichiers, données sensibles.

⌨️ 7.2 Input de type texte et mot de passe

Texte simple

<input type="text" 
       name="nom" 
       placeholder="Votre nom"
       maxlength="50">

Mot de passe

<input type="password" 
       name="mdp" 
       placeholder="Votre mot de passe"
       minlength="8">

Attributs communs aux inputs :

Attribut Description Exemple
nameNom du champ (clé lors de l'envoi)name="email"
placeholderTexte indicatif dans le champplaceholder="exemple@mail.com"
valueValeur par défautvalue="Jean"
requiredChamp obligatoirerequired
disabledChamp désactivédisabled
readonlyLecture seulereadonly
maxlengthNombre max de caractèresmaxlength="10"
minlengthNombre min de caractèresminlength="3"
autofocusFocus automatiqueautofocus

🏷️ 7.3 Les labels

La balise <label> associe un texte descriptif à un champ. Améliore l'accessibilité et l'expérience utilisateur (cliquer sur le label focus le champ).

Méthode 1 : for + id

<label for="nom">Nom :</label>
<input type="text" id="nom" name="nom">

L'attribut for du label doit correspondre à l'id de l'input.

Méthode 2 : Label englobant

<label>
    Nom :
    <input type="text" name="nom">
</label>

Le champ est à l'intérieur du label.

🎯 Exemple complet :

<form>
    <p>
        <label for="prenom">Prénom :</label>
        <input type="text" id="prenom" name="prenom" placeholder="Jean">
    </p>
    
    <p>
        <label for="email">Email :</label>
        <input type="email" id="email" name="email" required>
    </p>
</form>

🔘 7.4 Les boutons

Il existe plusieurs façons de créer des boutons :

<button>

<button type="submit">
    Envoyer
</button>

Peut contenir du HTML (icônes, images).

<input type="submit">

<input type="submit" 
       value="Envoyer">

Simple, texte seulement.

<input type="reset">

<input type="reset" 
       value="Effacer">

Remet le formulaire à son état initial.

Types de button :

  • type="submit" (défaut) : envoie le formulaire
  • type="reset" : réinitialise le formulaire
  • type="button" : bouton simple, sans action par défaut (pour JavaScript)

📄 7.5 Zone de texte multiligne (textarea)

Pour les messages longs, commentaires, etc.

<label for="message">Votre message :</label>
<textarea id="message" 
          name="message" 
          rows="5" 
          cols="40"
          placeholder="Écrivez votre message ici..."></textarea>

Attributs de textarea :

  • rows : nombre de lignes visibles
  • cols : nombre de colonnes (caractères par ligne)
  • wrap : gestion des retours à la ligne (soft/hard)
Astuce : Contrairement à input, textarea a une balise fermante et le contenu par défaut se met entre les balises, pas dans value.

🏗️ Les balises sémantiques HTML5

Structure de page, accessibilité, SEO

🔍 8.1 Qu'est-ce que la sémantique ?

La sémantique en HTML signifie donner du sens au contenu, pas seulement définir son apparence.

🎯
Pourquoi la sémantique ?
  • ✅ Meilleur référencement (SEO) - Google comprend mieux votre page
  • ✅ Accessibilité - Les lecteurs d'écran naviguent mieux
  • ✅ Maintenabilité - Code plus lisible pour les développeurs
  • ✅ Compatibilité future - Standards du web

Avant HTML5 : tout en <div>

<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="footer">...</div>

Avec HTML5 : balises sémantiques

<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

📋 8.2 Les différentes balises sémantiques

Balise Description Utilisation
<header> En-tête d'une page ou d'une section Logo, titre principal, menu
<nav> Menu de navigation principal Liens vers les pages du site
<main> Contenu principal unique Un seul par page, contenu central
<article> Contenu indépendant Article de blog, actualité, commentaire
<section> Section thématique Chapitre, partie d'article
<aside> Contenu complémentaire Barre latérale, publicité, liens connexes
<footer> Pied de page Copyright, contact, liens légaux
<address> Informations de contact Email, téléphone, adresse
<time> Date ou heure Date de publication, événement
<figure> / <figcaption> Illustration avec légende Images, graphiques, code
<mark> Texte surligné Mettre en évidence

🏛️ 8.3 Structure sémantique complète

<body>
    <header>
        <h1>Titre du site</h1>
        <nav>
            <ul>
                <li><a href="/">Accueil</a></li>
                <li><a href="/blog">Blog</a></li>
                <li><a href="/contact">Contact</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <header>
                <h2>Titre de l'article</h2>
                <p>Publié le <time datetime="2026-03-16">16 mars 2026</time></p>
            </header>

            <section>
                <h3>Introduction</h3>
                <p>Contenu de l'introduction...</p>
            </section>

            <section>
                <h3>Développement</h3>
                <p>Contenu principal...</p>
                
                <figure>
                    <img src="image.jpg" alt="Description">
                    <figcaption>Légende de l'image</figcaption>
                </figure>
            </section>

            <footer>
                <p>Écrit par Jean</p>
            </footer>
        </article>

        <aside>
            <h3>Articles récents</h3>
            <ul>
                <li><a href="#">Article 1</a></li>
                <li><a href="#">Article 2</a></li>
            </ul>
        </aside>
    </main>

    <footer>
        <p>&copy; 2026 Ambanivolo mi-DEV</p>
        <address>
            Contact : <a href="mailto:contact@example.com">contact@example.com</a>
        </address>
    </footer>
</body>

♿ 8.4 Accessibilité avec les balises sémantiques

Les balises sémantiques améliorent considérablement l'accessibilité pour les personnes utilisant des lecteurs d'écran.

♿
Rôles ARIA implicites :
  • <nav> a un rôle navigation implicite
  • <main> a un rôle main implicite
  • <article> a un rôle article implicite
  • <header> a un rôle banner (si dans body)
  • <footer> a un rôle contentinfo (si dans body)

Bonnes pratiques d'accessibilité :

  • Utilisez un seul <main> par page
  • Hiérarchie des titres correcte (h1 → h2 → h3...)
  • Navigation clavier possible (tabindex)
  • Ajoutez des aria-label si nécessaire

🎵 Le multimédia

Audio, vidéo, iframe et intégrations

🔊 9.1 La balise <audio>

HTML5 permet d'intégrer de l'audio directement sans plugins externes.

<audio controls>
    <source src="musique.mp3" type="audio/mpeg">
    <source src="musique.ogg" type="audio/ogg">
    Votre navigateur ne supporte pas l'élément audio.
</audio>

Attributs de <audio> :

Attribut Description
controlsAffiche les contrôles (play, pause, volume)
autoplayLecture automatique (souvent bloqué par navigateurs)
loopRépète en boucle
mutedDésactive le son
preloadCharge avant lecture (auto, metadata, none)

🎯 Exemple avec plusieurs sources :

<audio controls preload="metadata">
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    <source src="audio.wav" type="audio/wav">
    <!-- Message si aucun format supporté -->
    <p>Votre navigateur ne supporte pas l'audio. <a href="audio.mp3">Télécharger</a></p>
</audio>

🎬 9.2 La balise <video>

Intégration de vidéos directement dans la page.

<video width="640" height="360" controls poster="miniature.jpg">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    <source src="video.ogg" type="video/ogg">
    
    <!-- Sous-titres -->
    <track src="sous-titres.vtt" kind="subtitles" srclang="fr" label="Français">
    
    <!-- Message de secours -->
    <p>Votre navigateur ne supporte pas la vidéo. <a href="video.mp4">Télécharger</a></p>
</video>

Attributs spécifiques à la vidéo :

  • poster : image affichée avant la lecture
  • width / height : dimensions
  • controlslist : contrôle les contrôles affichés
  • disablepictureinpicture : désactive le mode picture-in-picture

Formats vidéo supportés :

  • MP4 (H.264) : meilleure compatibilité
  • WebM : format libre, bon pour le web
  • OGG : format libre, moins utilisé

🖼️ 9.3 La balise <iframe>

<iframe> permet d'intégrer une autre page HTML dans la page courante.

<iframe 
    src="https://example.com"
    width="800"
    height="600"
    title="Description"
    frameborder="0"
    allowfullscreen>
    <p>Votre navigateur ne supporte pas les iframes.</p>
</iframe>

Cas d'usage courants :

📺 Intégrer YouTube

<iframe 
    width="560"
    height="315"
    src="https://www.youtube.com/embed/VIDEO_ID"
    title="Vidéo YouTube"
    frameborder="0"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    allowfullscreen>
</iframe>

🗺️ Intégrer Google Maps

<iframe
    src="https://www.google.com/maps/embed?pb=!1m18!1m12!1m3!1d12345..."
    width="600"
    height="450"
    style="border:0;"
    allowfullscreen=""
    loading="lazy">
</iframe>

📄 Intégrer un PDF

<iframe
    src="document.pdf"
    width="100%"
    height="500px">
</iframe>

📱 Intégrer une page

<iframe
    srcdoc="<p>Contenu HTML direct</p>"
    width="100%"
    height="200px">
</iframe>

srcdoc permet d'injecter du HTML directement.

⚠️
Sécurité des iframes :

Utilisez l'attribut sandbox pour restreindre les capacités de l'iframe :

<iframe
    src="https://example.com"
    sandbox="allow-scripts allow-same-origin allow-forms">
</iframe>

Options : allow-scripts, allow-forms, allow-popups, allow-modals, etc.

📦 9.4 Embed et Object

Balises plus anciennes pour intégrer du contenu externe.

<embed>

<embed
    src="fichier.swf"
    type="application/x-shockwave-flash"
    width="400"
    height="300">

<object>

<object
    data="fichier.swf"
    type="application/x-shockwave-flash"
    width="400"
    height="300">
    Texte alternatif
</object>

Note : Ces balises sont moins utilisées aujourd'hui au profit de solutions natives HTML5.

📋 10.2 Listes déroulantes (select)

<label for="pays">Pays :</label>

    <option value="">Choisissez...</option>
    <optgroup label="Europe">

Attributs de select :

  • disabled : option non sélectionnable

<form>
    <label for="nom">Nom (3-20 lettres) :</label>
    <input type="text" id="nom" name="nom"
           minlength="3" maxlength="20" 
           pattern="[A-Za-z]+" 
           required>
    
    <label for="age">Âge (18-99) :</label>
    <input type="number" id="age" name="age"
           min="18" max="99" 
           step="1" 
           required>
    
    <button type="submit">Envoyer</button>
</form>

📦 10.5 Grouper les champs avec fieldset

<fieldset> permet de grouper des champs liés, avec une légende <legend>.

<fieldset>
    <legend>Informations personnelles</legend>
    
    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom">
    
    <label for="email">Email :</label>
    <input type="email" id="email" name="email">
</fieldset>

<fieldset disabled>
    <legend>Options désactivées</legend>
    <!-- Tous les champs ici sont désactivés -->
</fieldset>

Informations personnelles

📊 10.7 La balise output

<output> affiche le résultat d'un calcul.

<form oninput="result.value=parseInt(a.value)+parseInt(b.value)">
    <input type="range" id="a" value="50">
    +
    <input type="number" id="b" value="25">

    <output name="result" for="a b">75</output>

✍️ 10.8 Exercices pratiques

Exercice 1 : Formulaire d'inscription complet

Objectif : Créer un formulaire d'inscription avec validation.

Instructions :

  1. Champs : nom, prénom, email, mot de passe, confirmation
  2. Date de naissance (type date)
  3. Sexe (boutons radio)
  4. Pays (select avec optgroup)
  5. Centres d'intérêt (checkboxes)
  6. Couleur préférée (color)
  7. Niveau de compétence (range 0-100)
  8. Utilisez fieldset pour grouper
  9. Ajoutez des validations (required, minlength, etc.)

Exercice 2 : Calculateur simple

Objectif : Utiliser output pour des calculs.

Instructions :

  1. Créez un calculateur d'IMC (poids / taille²)
  2. Utilisez des inputs type range
  3. Affichez le résultat dans output

Exercice 3 : Formulaire de réservation

Objectif : Créer un formulaire complet.

Instructions :

  1. Date d'arrivée et de départ (date)
  2. Nombre de personnes (number)
  3. Type de chambre (select)
  4. Options supplémentaires (checkboxes)
  5. Demandes spéciales (textarea)
  6. Validation : date départ > date arrivée

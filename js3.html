🖥️ Manipulation du DOM

Interagir avec la page HTML

🔍 11.1 Qu'est-ce que le DOM ?

Le DOM (Document Object Model) est une représentation de la page HTML sous forme d'arbre d'objets. JavaScript peut manipuler cet arbre pour modifier dynamiquement la page.

📖 Visualisation :

                    document
                       │
                ┌──────┴──────┐
              html            │
               │              │
           ┌──┴──┐          body
         head    │            │
          │      │        ┌───┼───┐
        title   body     h1   p   div

🎯 11.2 Sélectionner des éléments

Méthode Description Exemple
getElementById() Sélection par ID (unique) document.getElementById("monId")
getElementsByClassName() Sélection par classe (collection) document.getElementsByClassName("maclasse")
getElementsByTagName() Sélection par balise (collection) document.getElementsByTagName("p")
querySelector() Sélection avec sélecteur CSS (premier) document.querySelector(".maclasse")
querySelectorAll() Sélection avec sélecteur CSS (tous) document.querySelectorAll("p.important") 
// HTML : <div id="app"><p class="texte">Bonjour</p></div>

// Par ID
const app = document.getElementById("app");

// Par classe
const textes = document.getElementsByClassName("texte");

// Par sélecteur CSS (recommandé)
const premier = document.querySelector(".texte");
const tous = document.querySelectorAll(".texte");

console.log(premier.textContent); // "Bonjour"
Astuce : Préférez querySelector() et querySelectorAll() car ils sont plus flexibles et utilisent la syntaxe CSS que vous connaissez déjà.

✏️ 11.3 Modifier le contenu

Propriété Description Exemple
textContent Texte seul (ignorer les balises) el.textContent = "Nouveau texte"
innerHTML HTML complet (peut créer des balises) el.innerHTML = "<strong>Gras</strong>"
innerText Texte visible (respecte CSS) el.innerText = "Texte visible" 
// HTML : <div id="demo">Ancien contenu</div>

const demo = document.getElementById("demo");

// Modifier le texte
demo.textContent = "Nouveau texte simple";

// Ajouter du HTML
demo.innerHTML = "<h2>Titre</h2><p>Paragraphe</p>";

// Ajouter du contenu sans effacer
demo.innerHTML += " <span>supplément</span>";
⚠️
Sécurité : Évitez innerHTML avec du contenu utilisateur (risque XSS). Préférez textContent pour le texte simple.

🖱️ Les événements

Réagir aux actions de l'utilisateur

🔍 12.1 Qu'est-ce qu'un événement ?

Un événement est une action ou une occurrence qui peut être détectée par JavaScript : clic, survol, frappe au clavier, chargement de page, etc.

👂 12.2 addEventListener()

La méthode moderne pour gérer les événements.

const bouton = document.querySelector("#monBouton");

bouton.addEventListener("click", function() {
    console.log("Bouton cliqué !");
});

// Avec fonction fléchée
bouton.addEventListener("click", () => {
    console.log("Encore cliqué !");
});

// Fonction nommée
function handleClick() {
    console.log("Clic détecté");
}
bouton.addEventListener("click", handleClick);

// Supprimer un écouteur
bouton.removeEventListener("click", handleClick);
💡
Avantages : On peut ajouter plusieurs écouteurs pour le même événement, et les supprimer facilement.

📋 12.3 Types d'événements courants

Catégorie Événements
Souris click, dblclick, mouseenter, mouseleave, mousemove
Clavier keydown, keyup, keypress
Formulaire submit, change, input, focus, blur
Document DOMContentLoaded, load, scroll, resize 
// Exemples
input.addEventListener("input", (e) => {
    console.log("Valeur actuelle:", e.target.value);
});

window.addEventListener("resize", () => {
    console.log("Fenêtre redimensionnée:", window.innerWidth);
});

document.addEventListener("keydown", (e) => {
    console.log(`Touche pressée: ${e.key}`);
});

📦 12.4 L'objet événement (e)

Le gestionnaire reçoit un paramètre contenant des informations sur l'événement.

bouton.addEventListener("click", (e) => {
    console.log(e.type);          // "click"
    console.log(e.target);        // L'élément qui a déclenché l'événement
    console.log(e.currentTarget); // L'élément avec l'écouteur
    console.log(e.clientX);       // Position X de la souris
    console.log(e.clientY);       // Position Y de la souris
});

input.addEventListener("keydown", (e) => {
    if (e.key === "Enter") {
        console.log("Entrée pressée");
    }
    
    if (e.ctrlKey && e.key === "s") {
        e.preventDefault();       // Empêche l'action par défaut (sauvegarde)
        console.log("Ctrl+S détecté");
    }
});

🌊 12.5 Propagation (bubbling et capturing)

Bubbling (remontée)

L'événement remonte du plus spécifique au plus général.

parent.addEventListener("click", () => {
    console.log("Parent cliqué");
});

enfant.addEventListener("click", () => {
    console.log("Enfant cliqué");
});
// Cliquer sur enfant affiche les deux

Arrêter la propagation

enfant.addEventListener("click", (e) => {
    e.stopPropagation(); // Empêche la remontée
    console.log("Seulement enfant");
});
// Empêcher le comportement par défaut
lien.addEventListener("click", (e) => {
    e.preventDefault(); // Empêche la navigation
    console.log("Lien cliqué mais pas de navigation");
});

💾 JSON et Stockage

Sauvegarder des données dans le navigateur

📋 13.1 JSON (JavaScript Object Notation)

JSON est un format léger pour échanger des données. Il ressemble aux objets JavaScript mais avec des règles plus strictes.

JavaScript JSON
{ nom: "Jean", age: 25 } {"nom":"Jean","age":25}
Les clés sans guillemets Les clés DOIVENT être en guillemets
Peut avoir des fonctions Uniquement des données 
// Convertir un objet JavaScript en JSON
const personne = {
    nom: "Jean",
    age: 25,
    adresse: {
        ville: "Antananarivo",
        pays: "Madagascar"
    }
};

const json = JSON.stringify(personne);
console.log(json); 
// "{\"nom\":\"Jean\",\"age\":25,\"adresse\":{\"ville\":\"Antananarivo\",\"pays\":\"Madagascar\"}}"

// Convertir JSON en objet JavaScript
const objet = JSON.parse(json);
console.log(objet.nom); // "Jean"

💿 13.2 localStorage (stockage persistant)

localStorage permet de stocker des données qui persistent même après fermeture du navigateur.

Méthode Description
localStorage.setItem(clé, valeur) Stocke une valeur
localStorage.getItem(clé) Récupère une valeur
localStorage.removeItem(clé) Supprime une clé
localStorage.clear() Vide tout le stockage 
// Stocker des données simples
localStorage.setItem("nom", "Jean");
localStorage.setItem("age", "25");

// Récupérer
const nom = localStorage.getItem("nom");
console.log(nom); // "Jean"

// Stocker des objets (besoin de JSON)
const utilisateur = { prenom: "Jean", age: 25 };
localStorage.setItem("user", JSON.stringify(utilisateur));

// Récupérer l'objet
const userData = JSON.parse(localStorage.getItem("user"));
console.log(userData.prenom); // "Jean"
⚠️
Important : localStorage ne stocke que des chaînes de caractères. Utilisez JSON.stringify() et JSON.parse() pour les objets/tableaux.

⏱️ 13.3 sessionStorage (temporaire)

sessionStorage fonctionne comme localStorage mais les données sont supprimées à la fermeture de l'onglet.

// Même API que localStorage
sessionStorage.setItem("temporaire", "Ceci disparaîtra");
sessionStorage.getItem("temporaire");
sessionStorage.removeItem("temporaire");
sessionStorage.clear();

🎯 13.4 Exemple : Thème sombre/clair

// Au chargement, appliquer le thème sauvegardé
const themeSauvegarde = localStorage.getItem("theme");

⏱️ Programmation asynchrone

Callbacks, Promises, async/await

🔍 14.1 Synchrone vs Asynchrone

Synchrone (bloquant)

Les instructions s'exécutent une par une, dans l'ordre. Si une instruction prend du temps, tout le reste attend.

console.log("1");
// Opération longue (bloquante)
console.log("2"); // Attend la fin

Asynchrone (non-bloquant)

Les opérations longues démarrent, mais le code continue. Une fonction de callback est appelée quand l'opération est terminée.

console.log("1");
setTimeout(() => {
    console.log("2");
}, 1000);
console.log("3"); // S'affiche avant 2

🎯 Résultat : 1, 3, 2 (après 1 seconde)

📞 14.2 Les callbacks

Un callback est une fonction passée en paramètre à une autre fonction, qui sera appelée quand une opération est terminée.

// Exemple avec setTimeout
console.log("Début");

setTimeout(function() {
    console.log("Ceci s'affiche après 2 secondes");
}, 2000);

console.log("Fin");

// Callback personnalisé
function chargerDonnees(callback) {
    console.log("Chargement...");
    setTimeout(() => {
        const donnees = { nom: "Jean", age: 25 };
        callback(donnees); // Appelle le callback avec les données
    }, 2000);
}

chargerDonnees(function(donnees) {
    console.log("Données reçues:", donnees);
});
⚠️
Callback Hell : L'utilisation excessive de callbacks imbriqués rend le code illisible (pyramide de la mort). C'est pourquoi Promises et async/await ont été créés.

🤝 14.3 Les Promises (Promesses)

Une Promise est un objet qui représente l'achèvement ou l'échec d'une opération asynchrone.

// Créer une Promise
const maPromise = new Promise((resolve, reject) => {
    // Opération asynchrone
    setTimeout(() => {
        const succes = true;
        
        if (succes) {
            resolve("Opération réussie !"); // Succès
        } else {
            reject("Échec de l'opération"); // Erreur
        }
    }, 2000);
});

// Utiliser la Promise
maPromise
    .then(resultat => {
        console.log("Succès:", resultat);
    })
    .catch(erreur => {
        console.error("Erreur:", erreur);
    })
    .finally(() => {
        console.log("Terminé (succès ou échec)");
    });
// Exemple plus concret : simulation de chargement
function chargerUtilisateur(id) {
    return new Promise((resolve, reject) => {
        console.log(`Chargement de l'utilisateur ${id}...`);
        
        setTimeout(() => {
            if (id === 1) {
                resolve({ id: 1, nom: "Jean" });
            } else {
                reject("Utilisateur non trouvé");
            }
        }, 1500);
    });
}

chargerUtilisateur(1)
    .then(utilisateur => console.log("Utilisateur:", utilisateur))
    .catch(erreur => console.error("Erreur:", erreur));

⛓️ 14.4 Chaînage de Promises

On peut enchaîner plusieurs opérations asynchrones.

function etape1() {
    return new Promise(resolve => {
        setTimeout(() => resolve("Étape 1 terminée"), 1000);
    });
}

function etape2(message) {
    console.log(message);
    return new Promise(resolve => {
        setTimeout(() => resolve("Étape 2 terminée"), 1000);
    });
}

function etape3(message) {
    console.log(message);
    return new Promise(resolve => {
        setTimeout(() => resolve("Étape 3 terminée"), 1000);
    });
}

etape1()
    .then(resultat1 => etape2(resultat1))
    .then(resultat2 => etape3(resultat2))
    .then(resultat3 => console.log(resultat3))
    .catch(erreur => console.error("Erreur:", erreur));

// Version simplifiée
etape1()
    .then(etape2)
    .then(etape3)
    .then(console.log)
    .catch(console.error);

✨ 14.5 async/await (ES2017)

async/await est une syntaxe plus lisible pour travailler avec les Promises.

// Fonction asynchrone avec async
async function chargerDonnees() {
    try {
        console.log("Chargement...");
        
        // await attend la résolution de la Promise
        const utilisateur = await chargerUtilisateur(1);
        console.log("Utilisateur chargé:", utilisateur);
        
        const commandes = await chargerCommandes(utilisateur.id);
        console.log("Commandes chargées:", commandes);
        
        return commandes;
    } catch (erreur) {
        console.error("Erreur lors du chargement:", erreur);
    }
}

// Appel de la fonction
chargerDonnees().then(resultat => {
    console.log("Terminé avec:", resultat);
});

// Ou avec await au niveau supérieur (dans un module)
// const resultat = await chargerDonnees();
💡
Bon à savoir : async/await est du sucre syntaxique sur les Promises. Le code est plus lisible et plus proche du synchrone, mais il faut gérer les erreurs avec try/catch.

⚡ 14.6 Promise.all (parallélisme)

Exécute plusieurs Promises en parallèle et attend qu'elles soient toutes résolues.

const p1 = chargerUtilisateur(1);
const p2 = chargerUtilisateur(2);
const p3 = chargerUtilisateur(3);

Promise.all([p1, p2, p3])
    .then(utilisateurs => {
        console.log("Tous les utilisateurs:", utilisateurs);
    })
    .catch(erreur => {
        console.error("Une des promesses a échoué:", erreur);
    });

// Avec async/await
async function chargerTous() {
    try {
        const utilisateurs = await Promise.all([
            chargerUtilisateur(1),
            chargerUtilisateur(2),
            chargerUtilisateur(3)
        ]);
        console.log("Tous chargés:", utilisateurs);
    } catch (erreur) {
        console.error("Erreur:", erreur);
    }
}

Chapitre 15 35 min Avancé

🌐 API Fetch

Faire des requêtes HTTP vers des serveurs

🔍 15.1 Qu'est-ce que l'API Fetch ?

Fetch est une API moderne pour faire des requêtes HTTP (GET, POST, etc.) et récupérer des ressources depuis le réseau.

C'est l'évolution de XMLHttpRequest (AJAX). Fetch retourne une Promise, ce qui le rend facile à utiliser avec async/await.

📥 15.2 Requête GET (récupérer des données)

// Avec Promises
fetch("https://jsonplaceholder.typicode.com/users/1")
    .then(reponse => {
        // Vérifier si la requête a réussi
        if (!reponse.ok) {
            throw new Error(`Erreur HTTP: ${reponse.status}`);
        }
        return reponse.json(); // Convertir en objet JavaScript
    })
    .then(donnees => {
        console.log("Utilisateur:", donnees);
    })
    .catch(erreur => {
        console.error("Erreur de fetch:", erreur);
    });

// Avec async/await (plus lisible)
async function chargerUtilisateur() {
    try {
        const reponse = await fetch("https://jsonplaceholder.typicode.com/users/1");
        
        if (!reponse.ok) {
            throw new Error(`Erreur ${reponse.status}`);
        }
        
        const utilisateur = await reponse.json();
        console.log("Utilisateur:", utilisateur);
    } catch (erreur) {
        console.error("Erreur:", erreur);
    }
}
Note : jsonplaceholder.typicode.com est une API gratuite pour tester. reponse.json() retourne une Promise, donc il faut await ou .then().

📄 15.3 Types de réponses

Méthode Description
reponse.json() Convertit en objet JavaScript (pour API JSON)
reponse.text() Récupère comme texte brut
reponse.blob() Pour les fichiers binaires (images)
reponse.formData() Pour les données de formulaire 
// Récupérer du texte
const reponseTexte = await fetch("https://example.com/texte.txt");
const texte = await reponseTexte.text();

// Récupérer une image
const reponseImage = await fetch("https://example.com/image.jpg");
const blob = await reponseImage.blob();
const url = URL.createObjectURL(blob);
document.querySelector("img").src = url;

📤 15.4 Requête POST

Pour envoyer des données au serveur (créer une ressource).

async function creerUtilisateur() {
    const nouvelUtilisateur = {
        nom: "Jean Dupont",
        email: "jean@example.com"
    };

    try {
        const reponse = await fetch("https://jsonplaceholder.typicode.com/users", {
            method: "POST",                           // Méthode HTTP
            headers: {
                "Content-Type": "application/json"    // Type de données envoyées
            },
            body: JSON.stringify(nouvelUtilisateur)   // Convertir en JSON
        });

        if (!reponse.ok) {
            throw new Error(`Erreur ${reponse.status}`);
        }

        const resultat = await reponse.json();
        console.log("Utilisateur créé:", resultat);
    } catch (erreur) {
        console.error("Erreur:", erreur);
    }
}

⚠️ 15.5 Gestion des erreurs

Fetch ne rejette la Promise que pour des erreurs réseau. Les erreurs HTTP (404, 500) doivent être vérifiées manuellement.

async function fetchSecurise(url) {
    try {
        const reponse = await fetch(url);
        
        // Vérifier les erreurs HTTP
        if (!reponse.ok) {
            throw new Error(`HTTP ${reponse.status}: ${reponse.statusText}`);
        }
        
        return await reponse.json();
    } catch (erreur) {
        // Erreur réseau ou HTTP
        console.error("Échec de la requête:", erreur.message);
        
        // Afficher un message à l'utilisateur
        document.querySelector("#erreur").textContent = 
            "Impossible de charger les données. Vérifiez votre connexion.";
        
        return null;
    }
}

🎯 15.6 Exemple : Afficher des utilisateurs

async function afficherUtilisateurs() {
    try {
        const reponse = await fetch("https://jsonplaceholder.typicode.com/users");
        const utilisateurs = await reponse.json();
        
        const liste = document.querySelector("#user-list");
        liste.innerHTML = ""; // Vider
        
        utilisateurs.forEach(user => {
            const li = document.createElement("li");
            li.innerHTML = `
                <strong>${user.name}</strong> (${user.email})
                <br>Ville: ${user.address.city}
            `;
            liste.appendChild(li);
        });
    } catch (erreur) {
        console.error("Erreur:", erreur);
    }
}

document.querySelector("#charger").addEventListener("click", afficherUtilisateurs);
